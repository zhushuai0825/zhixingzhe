# RAG 系统入门

RAG 是 Retrieval-Augmented Generation 的缩写，中文常称为检索增强生成。

一个典型的 RAG 系统包含五个核心步骤：

1. 文档解析：从 PDF、Markdown、网页或 txt 中提取文本。
2. 文本切片：把长文档拆分成适合检索和模型处理的小片段。
3. 向量化：使用 Embedding 模型把文本片段转换成向量。
4. 检索：根据用户问题找到最相关的文档片段。
5. 生成回答：把检索结果作为上下文交给大模型，让模型基于资料回答。

RAG 的关键价值是减少模型幻觉，让回答能够基于用户自己的知识库。

在产品设计中，RAG 问答最好展示引用来源。引用来源可以帮助用户判断回答是否可信，也能让用户快速回到原始资料。

MVP 阶段可以先使用简单文本切片和关键词检索跑通流程，后续再升级为向量数据库、Embedding 和 rerank。
