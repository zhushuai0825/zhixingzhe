# 功能需求清单

## 1. 工作台

### 1.1 功能描述

工作台是用户进入系统后的第一屏，用于展示近期知识库、最近文档、待办任务、学习进度和快捷入口。

### 1.2 页面元素

- 顶部导航
- 搜索框
- 快捷创建按钮
- 最近访问知识库
- 最近上传文档
- 今日任务
- 最近 AI 会话
- 系统状态提示

### 1.3 功能规则

- 默认展示最近 5 个知识库。
- 默认展示最近 5 个文档。
- 今日任务按优先级和截止时间排序。
- 当用户没有数据时，展示空状态和引导操作。

### 1.4 验收标准

- 新用户进入后能看到创建知识库、上传文档、开始问答的入口。
- 老用户进入后能快速回到最近知识库和今日任务。

## 2. 知识库管理

### 2.1 创建知识库

字段：

- 知识库名称，必填，最多 50 字。
- 描述，选填，最多 500 字。
- 图标，选填。
- 颜色，选填。

规则：

- 同一用户下知识库名称允许重复，但不推荐。
- 创建成功后进入知识库详情页。

### 2.2 知识库列表

展示字段：

- 名称
- 描述
- 文档数量
- 最近更新时间
- 最近问答时间

操作：

- 新建
- 搜索
- 编辑
- 删除
- 进入详情

### 2.3 删除知识库

规则：

- 删除前二次确认。
- 删除知识库会删除其下文档、切片、向量、会话关联。
- MVP 可采用硬删除。
- V1 建议改为软删除。

## 3. 文档管理

### 3.1 上传文档

支持格式：

- PDF
- Word docx
- txt
- Markdown

上传规则：

- MVP 单文件最大 20MB。
- 一次最多上传 5 个文件。
- 文件名最多 100 字。
- 不支持加密 PDF。
- 不保证扫描件 PDF，需要先经过 OCR。

处理流程：

1. 用户选择文件。
2. 前端校验格式和大小。
3. 后端保存原始文件。
4. 系统解析文本。
5. 系统生成文档摘要。
6. 系统切分文本。
7. 系统生成向量。
8. 文档状态变为可用。
9. 系统记录可用于后续 RAG 质量评估的片段、来源和相关度。

文档状态：

- 上传中
- 解析中
- 索引中
- 可用
- 失败

### 3.2 文档列表

展示字段：

- 文件名
- 类型
- 大小
- 状态
- 摘要
- 上传时间
- 更新时间

操作：

- 查看
- 重新解析
- 删除
- 生成摘要
- 加入问答上下文

### 3.3 文档详情

展示内容：

- 文档基础信息
- AI 摘要
- 关键词
- 主要知识点
- 原文预览
- 文档切片列表，开发调试时可见

## 4. AI 摘要

### 4.1 功能描述

系统对上传文档生成摘要，帮助用户快速理解文档内容。

### 4.2 输出内容

- 一句话摘要
- 详细摘要
- 核心观点
- 关键词
- 适合追问的问题

### 4.3 规则

- 10,000 字以内文档可直接摘要。
- 超长文档采用分段摘要再汇总。
- 摘要失败时允许重试。

## 5. 知识库问答

### 5.1 功能描述

用户可以选择一个知识库，基于该知识库中的文档进行问答。

### 5.2 输入

- 用户问题，必填，最多 2,000 字。
- 知识库 ID，必填。
- 可选文档范围。
- 可选模型。

### 5.3 输出

- AI 回答
- 引用来源
- 相关文档片段
- RAG 质量检查
- 后续追问建议
- 可执行操作

可执行操作：

- 保存为笔记
- 生成任务
- 继续追问
- 复制回答
- 点赞/点踩

### 5.4 回答规则

- 回答应优先基于检索内容。
- 引用来源至少包含文档名称和片段位置。
- 当检索结果不足时，应提示依据不足。
- 不允许伪造引用。

### 5.5 检索规则

MVP 推荐：

- 向量检索 Top 5。
- 可追加关键词检索。
- 根据相似度阈值过滤低相关片段。

后续优化：

- 混合检索
- rerank 重排序
- 按文档权限过滤
- 多轮上下文压缩

### 5.6 RAG 质量检查

功能描述：

系统在每次问答后返回基础评估结果，帮助学习者和测试工程师判断回答是否可靠。

输出字段：

- retrieved_count：检索到的片段数量。
- has_citations：是否存在引用来源。
- top_score：最高相关度。
- covered_terms：问题关键词中被证据覆盖的词。
- missing_terms：问题关键词中未被证据覆盖的词。
- verdict：grounded / weak_evidence / no_evidence。
- suggestion：下一步处理建议。

验收标准：

- 与知识库相关的问题应返回 grounded 或 weak_evidence，并展示引用。
- 与知识库无关的问题应返回 no_evidence 或 weak_evidence，并提示依据不足。
- 评估结果不得代替人工判断，只作为测试和学习辅助。

## 6. 聊天会话

### 6.1 功能描述

系统保存用户在知识库中的问答记录。

### 6.2 会话字段

- 会话标题
- 所属知识库
- 创建时间
- 更新时间
- 使用模型

### 6.3 消息字段

- 角色：user / assistant / system
- 消息内容
- 引用来源
- token 用量
- 调用耗时
- 创建时间

## 7. 任务管理

### 7.1 功能描述

用户可以手动创建任务，也可以从 AI 回答中生成任务。

### 7.2 任务字段

- 标题，必填，最多 100 字。
- 描述，选填。
- 状态：待办、进行中、已完成、已取消。
- 优先级：高、中、低。
- 截止时间，选填。
- 所属目标，选填。
- 所属知识库，选填。
- 来源：手动创建 / AI 生成。
- AI 生成原因，选填。

### 7.3 操作

- 新建
- 编辑
- 完成
- 取消
- 删除
- 设置优先级
- 设置截止时间

### 7.4 从 AI 回答生成任务

流程：

1. 用户点击「生成任务」。
2. 系统调用模型提取任务。
3. 前端展示待确认任务列表。
4. 用户可编辑标题、描述、优先级和截止时间。
5. 用户确认保存。

任务生成要求：

- 不生成过于空泛的任务。
- 每个任务应有明确动作。
- 如果回答不适合生成任务，应提示用户。

## 8. 学习目标

### 8.1 功能描述

用户可以创建学习目标，系统根据目标拆解计划和任务。

### 8.2 目标字段

- 目标名称
- 目标描述
- 开始日期
- 结束日期
- 每周可投入时间
- 关联知识库
- 当前状态

### 8.3 AI 生成计划

输入：

- 学习目标
- 时间周期
- 可用时间
- 用户基础
- 关联知识库摘要

输出：

- 阶段计划
- 周计划
- 每日任务建议
- 阶段检查点

MVP 可暂不实现完整学习目标模块，但数据结构应预留。

## 9. 复盘模块

### 9.1 功能描述

用户记录每日或每周复盘，AI 生成总结和改进建议。

### 9.2 复盘字段

- 复盘类型：日复盘 / 周复盘 / 月复盘。
- 完成事项。
- 未完成事项。
- 卡点。
- 情绪状态。
- 明日/下周计划。
- AI 建议。

### 9.3 AI 输出

- 总结
- 进度判断
- 风险提醒
- 下一步建议
- 计划调整建议

## 10. 模型配置

### 10.1 功能描述

用户可以配置不同模型服务商的 API Key，并选择默认模型。

### 10.2 支持服务商

MVP：

- DeepSeek
- 千问/Qwen OpenAI 兼容接口

后续：

- OpenAI
- Claude
- Gemini
- 本地模型

### 10.3 配置字段

- 服务商名称
- API Base URL
- API Key
- 默认模型
- 是否启用

### 10.4 安全要求

- API Key 不在前端明文展示。
- 后端存储时应加密。
- 日志中禁止打印完整 API Key。
