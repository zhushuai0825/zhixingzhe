# 页面与交互需求

## 1. 信息架构

一级导航建议：

- 工作台
- 知识库
- 问答
- 任务
- 目标
- 复盘
- 设置

MVP 可先保留：

- 工作台
- 知识库
- 任务
- 设置

## 2. 工作台页面

### 2.1 页面目标

让用户快速进入最近工作，并看到自己今天需要做什么。

### 2.2 页面结构

1. 顶部区域
   - 全局搜索
   - 新建知识库
   - 上传文档
   - 快速提问

2. 今日区域
   - 今日任务
   - 最近 AI 建议
   - 学习进度

3. 知识区域
   - 最近知识库
   - 最近文档
   - 最近会话

### 2.3 空状态

新用户空状态应提供 3 个主操作：

- 创建知识库
- 上传第一份文档
- 查看示例知识库

## 3. 知识库列表页

### 3.1 页面目标

帮助用户管理不同主题的知识空间。

### 3.2 交互

- 点击卡片进入知识库详情。
- 支持按名称搜索。
- 支持按最近更新排序。
- 删除知识库时弹出确认框。

### 3.3 卡片信息

- 知识库名称
- 描述
- 文档数量
- 最近更新时间
- 最近一次提问时间

## 4. 知识库详情页

### 4.1 页面目标

展示知识库内容，并提供上传、问答、摘要入口。

### 4.2 页面布局

左侧：

- 知识库信息
- 文档列表
- 标签或筛选

右侧：

- 问答区
- 文档摘要
- 最近会话

MVP 可以采用上下布局：

- 上方知识库信息和操作按钮
- 中间文档列表
- 下方问答入口

### 4.3 主要操作

- 上传文档
- 开始问答
- 编辑知识库
- 删除知识库

## 5. 文档上传交互

### 5.1 上传方式

- 点击上传按钮选择文件。
- 支持拖拽上传，V1 实现。

### 5.2 上传状态

状态应明确展示：

- 等待上传
- 上传中
- 解析中
- 建立索引中
- 完成
- 失败

失败时展示：

- 失败原因
- 重试按钮
- 删除按钮

## 6. 问答页面

### 6.1 页面目标

让用户基于知识库自然提问，并获得可信回答。

### 6.2 页面结构

左侧：

- 会话列表
- 新建会话

中间：

- 消息区域
- 输入框

右侧：

- 引用来源
- 相关文档片段
- 推荐追问

MVP 可简化为：

- 上方选择知识库
- 中间消息流
- 下方输入框
- 回答下方展示引用来源

### 6.3 消息交互

用户消息：

- 展示问题原文
- 支持复制

AI 消息：

- 展示回答正文
- 展示引用来源
- 支持复制
- 支持保存为笔记
- 支持生成任务
- 支持点赞/点踩

### 6.4 输入框

要求：

- 支持多行输入。
- Enter 发送，Shift + Enter 换行。
- 发送中禁用重复发送。
- 模型调用失败后允许重试。

## 7. 引用来源展示

### 7.1 展示内容

- 文档名称
- 片段序号或页码
- 命中片段摘要
- 相似度，调试模式可见

### 7.2 交互

- 点击引用来源，打开文档详情并定位到相关片段。
- 如果暂不能定位页码，至少展示原文片段。

## 8. 任务页面

### 8.1 页面目标

管理用户从知识和计划中转化出来的行动。

### 8.2 视图

MVP：

- 列表视图

V1：

- 看板视图
- 今日视图
- 日历视图

### 8.3 筛选

- 状态
- 优先级
- 所属知识库
- 来源
- 截止时间

### 8.4 任务卡片

展示：

- 标题
- 状态
- 优先级
- 截止时间
- 来源
- 关联知识库

操作：

- 完成
- 编辑
- 删除

## 9. 生成任务确认弹窗

### 9.1 触发位置

- AI 回答下方「生成任务」按钮。
- 学习计划页面「拆解为任务」按钮。

### 9.2 弹窗内容

- AI 生成的任务列表
- 每条任务可编辑
- 可删除不需要的任务
- 可批量保存

### 9.3 规则

- 任务为空时提示无法生成。
- 保存前必须至少有一条任务。
- 标题为空的任务不能保存。

## 10. 设置页面

### 10.1 模型设置

字段：

- 服务商
- API Base URL
- API Key
- 默认模型
- 测试连接按钮

### 10.2 知识库设置

字段：

- 默认切片长度
- 默认重叠长度
- 检索 Top K
- 是否展示相似度

MVP 可使用系统默认值，暂不开放全部配置。
