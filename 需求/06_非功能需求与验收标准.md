# 非功能需求与验收标准

## 1. 性能需求

### 1.1 页面性能

- 首屏加载时间应小于 3 秒，本地开发环境除外。
- 知识库列表在 100 个知识库以内应流畅展示。
- 文档列表在 1,000 个文档以内应支持分页或虚拟列表。

### 1.2 文档处理性能

MVP 目标：

- 10MB 以内文本型 PDF，60 秒内完成解析和索引。
- 5MB 以内 Markdown/txt，20 秒内完成解析和索引。
- 单个知识库 500 个切片以内，问答检索应在 2 秒内完成。

### 1.3 AI 响应性能

- 问答请求应支持流式输出。
- 非流式场景下，10 秒内无响应应展示等待状态。
- 超过 60 秒应提示超时并允许重试。

## 2. 稳定性需求

- 文档处理失败不能影响其他文档。
- 模型调用失败不能导致页面崩溃。
- 后端异常需要返回统一错误格式。
- 长任务需要记录处理状态。
- 系统重启后，已上传文档和任务不丢失。

统一错误格式建议：

```json
{
  "code": "MODEL_CALL_FAILED",
  "message": "模型调用失败，请检查 API Key 或稍后重试",
  "detail": "可选调试信息"
}
```

## 3. 安全需求

### 3.1 API Key 安全

- API Key 不能出现在前端页面源码中。
- API Key 不允许写入普通日志。
- API Key 数据库存储必须加密。
- 用户只能使用自己的模型配置。

### 3.2 文件安全

- 上传文件需校验扩展名和 MIME 类型。
- 不执行用户上传文件中的任何脚本。
- 文件访问需要校验用户权限。
- 删除文档时同步删除解析文本和索引。

### 3.3 数据隔离

V1 多用户模式下：

- 用户只能访问自己的知识库、文档、任务、会话。
- 所有查询必须带 user_id 过滤。
- 管理员能力暂不开放。

## 4. 可用性需求

- 新用户 3 分钟内应能完成首次文档上传和问答。
- 关键按钮文案应清晰，避免使用技术术语。
- 文档解析中需要展示进度或状态。
- 空状态需要引导下一步操作。
- 错误提示要告诉用户如何处理。

## 5. 可维护性需求

- 模型供应商需要通过统一接口适配。
- 提示词模板需要集中管理，不能散落在业务代码中。
- 文档解析、切片、向量化、问答生成应拆分为独立模块。
- 关键业务逻辑应有单元测试。
- 接口需要有基础文档。

## 6. 可观测性需求

需要记录：

- 接口请求日志
- 文档处理日志
- 模型调用日志
- 错误日志
- token 用量
- 响应耗时

需要能定位：

- 哪个文档解析失败
- 哪个模型调用失败
- 哪个知识库检索无结果
- 哪次回答没有引用来源

## 7. 兼容性需求

MVP 浏览器支持：

- Chrome 最新两个主版本
- Edge 最新两个主版本

移动端：

- MVP 做基础响应式适配
- V1 优化移动端体验

## 8. 验收用例

### 8.1 知识库创建

前置条件：用户进入系统。

步骤：

1. 点击新建知识库。
2. 输入名称和描述。
3. 点击保存。

预期结果：

- 创建成功。
- 跳转到知识库详情页。
- 列表中可看到新知识库。

### 8.2 文档上传成功

前置条件：存在一个知识库。

步骤：

1. 进入知识库详情页。
2. 上传一个 txt 文件。
3. 等待处理完成。

预期结果：

- 文档状态从上传中变为可用。
- 文档详情中展示摘要。
- 系统生成切片。

### 8.3 文档上传失败

步骤：

1. 上传一个不支持的文件格式。

预期结果：

- 前端阻止上传或后端返回错误。
- 页面展示明确错误提示。

### 8.4 知识库问答成功

前置条件：知识库中已有可用文档。

步骤：

1. 输入与文档相关的问题。
2. 点击发送。

预期结果：

- 系统返回回答。
- 回答下方展示引用来源。
- 会话记录被保存。

### 8.5 无依据问题处理

前置条件：知识库中没有相关内容。

步骤：

1. 提问与知识库无关的问题。

预期结果：

- 系统提示当前知识库中没有找到足够依据。
- 不应编造引用来源。

### 8.6 从回答生成任务

前置条件：存在一条 AI 回答。

步骤：

1. 点击生成任务。
2. 查看生成任务弹窗。
3. 编辑其中一条任务。
4. 点击保存。

预期结果：

- 任务保存成功。
- 任务列表可查看。
- 任务来源标记为 AI 生成。

### 8.7 模型配置测试

步骤：

1. 进入设置页。
2. 填写 API Base URL、API Key、默认模型。
3. 点击测试连接。

预期结果：

- 配置正确时提示连接成功。
- 配置错误时提示失败原因。
- 页面不展示完整 API Key。

## 9. 发布检查清单

- 核心链路可用：创建知识库 -> 上传文档 -> 问答 -> 生成任务。
- 异常链路可用：无模型配置、上传失败、模型失败、无检索结果。
- API Key 不出现在日志和前端。
- 数据库迁移脚本可重复执行。
- README 中包含启动方式。
- 至少有一组示例文档用于演示。
