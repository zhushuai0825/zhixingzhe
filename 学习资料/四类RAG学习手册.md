# 四类 RAG 学习手册

这份手册只保留你现在最应该理解的四类 RAG：

```text
Naive RAG -> Advanced RAG -> Agentic RAG -> Graph RAG
```

学习顺序建议：

1. 先用 Naive RAG 跑通主链路。
2. 再把 Advanced RAG 作为当前重点深入。
3. 然后学习 Agentic RAG，把系统做成能自主决策的行动助手。
4. 最后把 Graph RAG 当作专项能力学习。

不要一开始同时学太多分类。Standard RAG 基本可以归入 Naive RAG；Hybrid RAG、Fusion RAG、Adaptive RAG 里的很多能力可以放进 Advanced RAG；Multi-Modal、Structured、Code RAG 是按知识形态划分，后续可以作为扩展方向，不作为当前主线。

---

## 1. Naive RAG：基础 / 标准 RAG

### 1.1 它是什么

Naive RAG 是最基础的 RAG。

它的流程是：

```text
文档 -> 分块 -> 向量化 -> 向量检索 -> 拼接上下文 -> 大模型生成回答
```

简单理解：

```text
先把文档切成小块，再把问题和文档块都变成向量，找最相似的文档块，最后让大模型基于这些文档块回答。
```

### 1.2 它解决什么问题

普通大模型直接回答时，可能凭记忆编答案。Naive RAG 给模型加了一步“查资料”：

```text
用户问题 -> 查知识库 -> 拿到相关片段 -> 基于片段回答
```

这样可以让回答更贴近你的文档。

### 1.3 它的优点

- 最简单，最适合入门。
- 工程链路清晰。
- 容易部署。
- 适合 FAQ、简单文档问答、产品说明问答。

### 1.4 它的缺点

- 只靠向量检索，可能找不准。
- 问题写得不好时，检索结果容易偏。
- 如果文档切片不好，答案可能断章取义。
- 没有重排序和评测时，容易看起来能答，实际依据不强。

### 1.5 在知行者里对应什么

知行者现在的基础链路就是 Naive RAG 的主干：

```text
上传文档
-> 文档解析
-> 文本切片
-> Embedding
-> FAISS 向量检索
-> 拼接引用
-> 生成回答
```

你要在系统里重点看：

- 文档上传
- 文档切片
- RAG 实验室
- AI 问答
- 引用来源

### 1.6 你怎么练

在 RAG 实验室里这样做：

```text
问题：RAG 的核心流程是什么？
chunk_size：600
overlap：120
top_k：5
Hybrid：先打开
Rerank：先打开
```

然后观察：

- 命中了几个片段。
- 最高相关度是多少。
- 命中的片段是否真的包含答案。
- 引用是不是来自正确文档。

你能解释这些，就说明 Naive RAG 主链路已经入门。

## 2. Advanced RAG：进阶 / 企业级 RAG

### 2.1 它是什么

Advanced RAG 是在 Naive RAG 基础上做全流程优化。

它不是一个单独步骤，而是一组优化方法：

```text
检索前优化 -> 检索中优化 -> 检索后优化 -> 生成与评测优化
```

### 2.2 检索前优化

检索前做的是：让问题更适合检索。

常见能力：

- 查询重写：把口语问题改成更适合搜索的问题。
- 查询扩展：补充同义词、英文名、缩写。
- 意图识别：判断用户是问定义、流程、对比、配置、有没有依据。

例子：

```text
用户问：这个怎么弄？
系统改写：RAG 系统如何完成文档上传、切片、向量化和检索？
```

### 2.3 检索中优化

检索中做的是：用更多方式找证据。

常见能力：

- 向量检索：看语义相似。
- BM25：看关键词命中。
- Hybrid 检索：向量 + BM25。
- 多路召回：从不同索引、不同切片策略、不同模型召回结果。

知行者现在已经有：

```text
Hybrid = 向量分 * 0.65 + BM25 分 * 0.35
```

### 2.4 检索后优化

检索后做的是：把召回结果变得更干净、更可靠。

常见能力：

- Rerank：把候选片段重新排序。
- 上下文压缩：只保留和问题有关的句子。
- 去冗余：删除重复片段。
- 引用筛选：只给模型真正有价值的证据。

知行者现在已经有轻量 Rerank：

```text
rerank_score = base_score * 0.65 + token_coverage * 0.25 + completeness * 0.10
```

### 2.5 它的优点

- 更接近企业真实 RAG。
- 检索更准。
- 更容易处理复杂问题。
- 可以通过评测集持续优化。

### 2.6 它的缺点

- 参数更多。
- 链路更长。
- 需要评测，否则不知道优化是不是真的有效。
- 可能因为过度处理导致成本变高。

### 2.7 在知行者里对应什么

知行者现在已经具备 Advanced RAG 的雏形：

- BM25 + 向量混合检索
- Rerank 重排序
- RAG 实验室
- 自动评测集
- 无依据拒答
- 引用质量检查

### 2.8 你怎么练

固定同一个问题，每次只改一个变量：

```text
第一轮：Hybrid 开，Rerank 开
第二轮：Hybrid 关，Rerank 开
第三轮：Hybrid 开，Rerank 关
第四轮：Top K 从 5 改到 8
第五轮：chunk_size 从 600 改到 1200
```

每次观察：

- 命中片段是否更准。
- 是否出现重复片段。
- 是否把答案切断。
- 评测集通过率是否变化。

Advanced RAG 是你现在最应该重点学的部分。

## 3. Agentic RAG：智能体 RAG

### 3.1 它是什么

Agentic RAG 是让大模型像智能体一样使用 RAG。

Naive RAG 通常是：

```text
一次检索 -> 一次生成
```

Agentic RAG 是：

```text
判断是否需要检索
-> 决定检索什么
-> 调用工具
-> 检查证据够不够
-> 不够就继续检索
-> 最后生成回答或任务
```

### 3.2 它适合什么

适合复杂问题和长任务，例如：

- 分析一份测试报告并生成风险清单。
- 阅读多份文档后生成学习计划。
- 根据知识库找问题原因，再生成修复任务。
- 多轮追问中动态补充检索。

### 3.3 它的优点

- 能处理复杂任务。
- 能动态决定是否继续检索。
- 能结合工具，例如搜索、数据库、任务系统。
- 更像真正的 AI 助手。

### 3.4 它的缺点

- 更难测试。
- 可能循环检索。
- 可能调用错工具。
- 成本更难控制。
- 必须有步骤日志、停止条件和评测方案。

### 3.5 在知行者里未来怎么做

知行者未来可以这样演进：

```text
用户提出目标
-> Agent 判断需要哪些资料
-> 检索知识库
-> 如果依据不足，继续检索或让用户补文档
-> 生成回答
-> 生成任务
-> 进入复盘
```

这会把知行者从“知识问答系统”升级成“知识行动助手”。

### 3.6 你现在怎么学

现在先不用写复杂 Agent 框架。

你先学会设计流程：

```text
1. 用户问题是什么？
2. 是否需要检索？
3. 应该检索哪个知识库？
4. 一次检索够不够？
5. 没有依据时要不要追问用户？
6. 结果是否要生成任务？
```

等 Naive 和 Advanced 扎实后，再做 Agentic RAG。

## 4. Graph RAG：图谱 RAG

### 4.1 它是什么

Graph RAG 是把文档里的知识抽取成图谱。

图谱由两部分组成：

```text
实体 + 关系
```

例子：

```text
RAG -> 包含 -> 文档解析
RAG -> 包含 -> Embedding
Embedding -> 用于 -> 语义检索
BM25 -> 用于 -> 关键词检索
```

这样系统不只是找相似片段，还能沿着关系做多跳推理。

### 4.2 它适合什么

适合关系复杂的场景：

- 金融风控
- 医疗知识关联
- 法律条文推理
- 公司组织关系
- 项目依赖分析
- 论文方法和任务关系分析

### 4.3 它的优点

- 适合多跳问题。
- 能解释实体之间的关系。
- 能回答“谁影响谁”“A 和 B 有什么关系”这类问题。

### 4.4 它的缺点

- 构建成本高。
- 实体抽取和关系抽取容易出错。
- 需要图数据库或图查询能力。
- 不适合你现在作为第一阶段主攻。

### 4.5 在知行者里未来怎么做

后期可以把 AI 前沿资料包做成图谱：

```text
论文 -> 提出方法 -> 解决问题
模型 -> 属于领域 -> 多模态
项目 -> 使用技术 -> Agent
技术 -> 适用场景 -> 企业知识库
```

然后系统可以回答：

```text
这些 Agent 项目之间有什么共同技术？
哪些论文和 RAG 评测有关？
某个模型适合解决什么问题？
```

### 4.6 你现在怎么学

现在先做到：

- 能识别实体。
- 能识别关系。
- 能画出简单实体关系图。
- 知道 Graph RAG 是后期专项，不是当前主线。

---

## 5. 你的最终学习路线

推荐路线：

```text
第 1 步：Naive RAG
目标：跑通上传、切片、向量化、检索、回答、引用。

第 2 步：Advanced RAG
目标：掌握 Hybrid、Rerank、评测集、拒答、上下文优化。

第 3 步：Agentic RAG
目标：让系统能判断、检索、反思、生成任务。

第 4 步：Graph RAG
目标：做实体关系、多跳推理、复杂知识关联。
```

当前最重要的是：

```text
Naive RAG 跑通 + Advanced RAG 深入
```

这两步学扎实，你就已经具备企业 RAG 项目的核心能力。
