四类 RAG 学习手册

只学最重要的四条主线：Naive RAG 跑通基础链路，Advanced RAG 做企业级优化，Agentic RAG 做自主决策，Graph RAG 做复杂关系推理。

Naive RAG Advanced RAG Agentic RAG Graph RAG 学习路线

为什么只保留这四类

RAG 分类很多，但对你现在来说，太多分类会分散注意力。Standard RAG 可以归到 Naive RAG；Hybrid、Fusion、Adaptive 的很多能力可以归到 Advanced RAG；Multi-Modal、Structured、Code 是知识形态扩展，后面再作为专项做。

当前重点：Naive RAG 跑通 + Advanced RAG 深入。这两步学扎实，就已经具备企业 RAG 项目的核心能力。

1. Naive RAG：基础 / 标准 RAG

文档 -> 分块 -> 向量化 -> 向量检索 -> 拼接上下文 -> 大模型生成回答
是什么

最基础的 RAG。先把文档切成小块，再把问题和文档块都变成向量，找最相似的片段，让大模型基于片段回答。

适合什么

FAQ、简单文档问答、产品说明问答、RAG 入门项目。

优点

简单、清晰、容易部署，是所有高级 RAG 的地基。

缺点

检索不准时容易答偏；没有重排序、查询改写和评测时，幻觉风险更高。

在知行者里怎么练

进入 RAG 实验室，用默认参数跑一次“RAG 的核心流程是什么？”。重点看命中片段是否包含答案、引用是否来自正确文档。

2. Advanced RAG：进阶 / 企业级 RAG

检索前：查询重写、扩展、意图识别
检索中：Hybrid 检索、多路召回
检索后：Rerank、上下文压缩、去冗余
评测：无依据拒答、自动评测集
是什么

在 Naive RAG 上做全流程优化，让检索更准、证据更干净、回答更可信。

适合什么

企业知识库、测试知识助手、智能客服、产品助手等需要较高准确率的场景。

优点

更接近工业界主流，能处理更多复杂问题，可以通过评测持续优化。

缺点

链路更长，参数更多，如果没有评测集，很难知道优化是否真的有效。

在知行者里怎么练

同一个问题只改一个变量：先对比 Hybrid 开关，再对比 Rerank 开关，最后运行自动评测集，看通过率是否变化。

3. Agentic RAG：智能体 RAG

判断是否需要检索 -> 决定检索什么 -> 调用工具 -> 检查证据是否够 -> 不够继续检索 -> 生成回答或任务
是什么

用大模型当智能体，动态决定是否检索、检索几次、用什么工具、是否反思修正。

适合什么

复杂多跳问题、长任务、自动分析报告、测试风险分析、需要工具调用的工作流。

优点

更像真正的 AI 助手，能把知识问答扩展成计划、任务和复盘。

缺点

更难测试，可能循环检索、调用错工具、成本失控，需要步骤日志和停止条件。

在知行者里未来怎么做

让 AI 自动拆解问题、选择知识库、检索资料、判断依据是否足够、生成任务，并进入复盘。它会把知行者从知识问答升级成知识行动助手。

4. Graph RAG：图谱 RAG

文档 -> 实体抽取 -> 关系抽取 -> 知识图谱 -> 图检索 / 多跳推理 -> 生成回答
是什么

把文档构建成实体和关系组成的知识图谱，再支持多跳关联推理。

适合什么

金融风控、医疗知识关联、法律条文推理、组织关系、论文方法关系分析。

优点

适合回答“谁影响谁”“A 和 B 有什么关系”“多份资料之间如何关联”。

缺点

构建成本高，实体和关系抽取容易出错，需要图数据库或图查询能力。

你现在怎么学

先不用实现。现在只要能从文档里识别实体和关系，并画出简单关系图。Graph RAG 是后期专项，不是当前主线。

最终学习路线

阶段学习目标知行者里怎么做
Naive RAG跑通上传、切片、向量化、检索、回答、引用。用 RAG 实验室默认参数跑通基础问题。
Advanced RAG掌握 Hybrid、Rerank、评测集、拒答、上下文优化。对比参数并运行自动评测集。
Agentic RAG让系统能判断、检索、反思、生成任务。设计问题拆解和工具调用流程。
Graph RAG做实体关系、多跳推理、复杂知识关联。先从 AI 前沿资料包抽取实体关系。
