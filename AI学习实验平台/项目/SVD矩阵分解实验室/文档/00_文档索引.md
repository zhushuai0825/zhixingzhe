# SVD 矩阵分解实验室文档索引

## 1. 一句话理解

SVD / 矩阵分解就是：把用户和物品都变成一组数字向量，用这两组数字的点积预测用户可能喜欢哪个物品。

## 2. 行为矩阵是什么

```text
          RAG   Embedding   ItemCF   SVD
小周       1       1          0      0
小林       1       1          0      0
小陈       0       0          1      1
```

行是用户，列是物品，值表示用户是否喜欢、点击、购买或评分。

## 3. 为什么矩阵稀疏

真实系统里物品非常多，用户只接触过很少一部分。所以矩阵里大多数位置是空的。推荐系统要做的事，就是根据已知位置预测未知位置。

## 4. 隐向量是什么

隐向量不是人工标签，而是算法从行为中学习出的低维表达。它可能混合了主题、难度、风格、人群偏好等因素。

教学版中为了好理解，把物品向量设成 2 维：

```text
RAG 入门: [0.82, 0.18]
SVD 矩阵分解: [0.18, 0.95]
```

真实系统可能是 32 维、64 维、128 维，且每一维不一定能直接解释成一个中文标签。

## 5. 点积怎么理解

```text
score(user, item) = user_vector · item_vector
```

如果用户向量和物品向量方向一致，点积通常更高。你可以把它理解成“用户兴趣”和“物品属性”的匹配程度。

## 6. 3D 可视化怎么看

- 用户节点：当前用户隐向量。
- 物品节点：候选物品隐向量。
- 连线越粗：点积分越高。
- 距离越近或方向越一致：越可能被推荐。

## 7. 面试题

### SVD 解决了什么问题？

它用低维隐向量去表示用户和物品，从稀疏行为矩阵里发现潜在兴趣关系，预测用户没接触过的物品是否可能喜欢。

### SVD 和 ItemCF/UserCF 有什么区别？

ItemCF/UserCF 直接基于共现或相似用户；SVD 会学习隐向量，不只是数共同喜欢。SVD 可以发现更潜在的关系，但解释性弱一些。

### 为什么说隐向量解释性弱？

因为隐向量的维度是算法学出来的，不一定对应“AI、推荐、测试”这种人工标签。它更像压缩后的兴趣空间。

### SVD 有什么冷启动问题？

新用户没有行为，学不到用户向量；新物品没有行为，学不到物品向量。通常要结合标签、内容、热门推荐或人工规则兜底。

## 8. 从 0 到 1 手算一遍

教学版里，我们先用 2 维向量理解点积。

假设当前用户向量是：

```text
user = [0.80, 0.20]
```

候选 A 是 AI/RAG 类内容：

```text
itemA = [0.90, 0.30]
```

候选 B 是推荐算法类内容：

```text
itemB = [0.20, 0.90]
```

计算点积：

```text
score(user, itemA) = 0.80 * 0.90 + 0.20 * 0.30
                   = 0.72 + 0.06
                   = 0.78

score(user, itemB) = 0.80 * 0.20 + 0.20 * 0.90
                   = 0.16 + 0.18
                   = 0.34
```

所以系统更可能推荐候选 A。

口语化理解：用户兴趣方向偏第一维，候选 A 也偏第一维，所以匹配度更高。

## 9. 真实训练大概是什么

教学版用“已喜欢物品向量平均值”模拟用户向量，这是为了让你看懂流程。真实矩阵分解通常会训练两个矩阵：

```text
用户矩阵 U: 每个用户一个向量
物品矩阵 V: 每个物品一个向量
预测分数: U[user] · V[item]
```

训练目标是让已知行为的预测分数尽量接近真实行为。例如用户喜欢过的物品分数应该高，没喜欢或负反馈物品分数应该低。

常见做法包括：

- 显式评分预测：例如预测 1 到 5 星评分。
- 隐式反馈建模：点击、观看、购买等行为代表偏好强弱。
- 加正则化：防止向量数值过大、过拟合。
- 加偏置项：用户有整体活跃度差异，物品有整体热门度差异。

## 10. 工程落地怎么做

真实推荐系统里，SVD 或矩阵分解通常这样落地：

1. 收集用户行为矩阵。
2. 把行为转成训练样本。
3. 训练用户向量和物品向量。
4. 离线保存物品向量。
5. 定期或实时更新用户向量。
6. 线上用用户向量和候选物品向量做点积。
7. 取 Top N 作为召回结果。
8. 再进入排序模型、规则过滤和多样性控制。

## 11. 常见误区

- 误区一：隐向量就是标签。隐向量是算法学出来的，不一定能人工命名。
- 误区二：SVD 一定比协同过滤好。数据少、冷启动强时，标签和热门可能更稳。
- 误区三：点积高就一定推荐。还要考虑已看过滤、业务规则、质量、安全、多样性。
- 误区四：教学 2 维等于真实生产。真实生产维度更多，而且需要训练、评估和更新。
- 误区五：矩阵分解只适合评分。点击、播放、购买等隐式反馈也可以建模。

## 12. 面试回答模板

如果面试官问“SVD 推荐怎么理解”，可以这样回答：

```text
SVD 或矩阵分解会把用户-物品行为矩阵分解成用户隐向量和物品隐向量。
用户向量表示用户的潜在兴趣，物品向量表示物品的潜在属性。
推荐时用用户向量和候选物品向量做点积，点积分越高，预测用户越可能喜欢。
它相比 ItemCF/UserCF 能学习更潜在的兴趣关系，但解释性弱，新用户和新物品仍然有冷启动问题。
工程上通常离线训练向量，线上用点积做快速召回，再交给排序模型处理。
```

## 14. 动手实验建议

建议你做两个对比：

1. 新增一个偏 AI/RAG 的物品，例如 `Graph RAG 实战课`，标签填 `AI, RAG, 图谱`。
2. 新增一个偏推荐算法的物品，例如 `召回排序实战课`，标签填 `推荐, 算法, 排序`。
3. 给当前用户多添加 AI/RAG 类喜欢，观察用户向量第一维是否更高。
4. 再切换到推荐算法兴趣更强的用户，观察第二维是否更高。

你要重点看两件事：用户向量从喜欢行为来，推荐分从用户向量和物品向量点积来。
