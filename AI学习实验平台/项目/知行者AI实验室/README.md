# 知行者 AI 实验室

知行者 AI 实验室是一个面向 AI 应用开发学习的实战项目。

项目目标不是做一个杂乱的大而全系统，而是围绕三个核心能力逐步建设：

1. RAG 实验室：学会企业知识库问答的完整工程链路。
2. Agent 实验室：学会让 AI 拆任务、调工具、检查结果、生成行动。
3. 可视化实验台：看见数据如何入库、切片如何生成、向量如何检索、Agent 如何运行。

最终系统会覆盖测试工程师转 AI 应用工程师需要掌握的主要知识：

- Python
- FastAPI
- PostgreSQL
- 文档解析
- 文本切片
- Embedding
- Chroma / FAISS / Milvus
- BM25
- Rerank
- RAG 评测
- 大模型 API
- Tool Calling
- Agent 状态管理
- LangChain / LlamaIndex / LangGraph
- Ollama 本地模型
- Docker / 云服务器部署

## 当前阶段

当前已经进入第一版稳定闭环：

- Docker Desktop 运行 PostgreSQL 和 Chroma。
- FastAPI 后端可以创建知识库、导入文本、切片、写入 PostgreSQL、写入 Chroma、查询 Top K。
- 前端原型已经接入后端，可以在页面里做真实 RAG 入库、Chroma 向量检索、BM25 关键词检索、Hybrid RAG 融合检索和大模型基于证据生成回答。
- Agent 实验室已经接真实后端，可以围绕目标调用 RAG、展示大模型最终回答、展开引用来源、生成任务并保留运行轨迹。

本地访问：

```text
前端：http://127.0.0.1:8088
后端接口文档：http://127.0.0.1:8010/docs
```

## 当前怎么用

1. 打开前端 `http://127.0.0.1:8088`。
2. 进入 `RAG 实验室`，点击 `创建知识库`。
3. 粘贴文本，或选择本地 `.txt/.md/.pdf` 文件。
4. 点击 `导入并切片`，观察解析文本、切片、写 PostgreSQL、写 Chroma 的步骤状态。
5. 进入 `可视化实验台`，输入问题，选择 `Hybrid / Vector / Keyword` 检索模式并点击 `真实检索`。
6. 调整 `最低相关度`，观察系统什么时候依据充分、什么时候应该拒答。
7. 查看检索解释面板：问题向量预览、Chroma 查询参数、BM25 查询、Top K 排名、依据判断、生成模型、拼接进 Prompt 的上下文。
8. 进入 `Agent 实验室`，输入目标，先看 `Agent 最终结果`，再看工具调用轨迹和步骤调试详情。

## 已完成能力

- 本地 Docker 运行 PostgreSQL 和 Chroma。
- FastAPI 后端。
- 知识库创建和列表。
- 文本、Markdown、TXT、PDF 导入。
- PDF 文本解析，暂不支持扫描版 OCR。
- 文本切片和重叠切片。
- 本地学习版 Embedding。
- Chroma 向量写入和 Top K 检索。
- 学习版 BM25 关键词检索。
- Hybrid RAG：向量分 + BM25 分融合排序。
- PostgreSQL 保存文档、切片和检索记录。
- 前端真实 RAG 入库实验。
- 前端真实检索实验。
- SenseAudio 兼容大模型生成回答，默认使用 `deepseek-v4-pro`。
- 检索解释面板。
- 最低相关度阈值和依据不足拒答。
- Agent 最终结果展示。
- Agent 引用来源展开查看原文。
- Agent 工具调用轨迹。
- Agent 任务生成和反思检查。
- 数据中心删除知识库和文档。
- 3D 向量空间展示、旋转、缩放和命中高亮。
