# 产品总需求：知行者 AI 实验室

## 1. 产品定位

知行者 AI 实验室是一个帮助测试工程师系统学习 AI 应用开发的实验型产品。

它不是普通笔记软件，也不是单纯聊天机器人，而是一个可以把 AI 技术拆开、运行、观察、评测和沉淀的学习工作台。

一句话定位：

> 通过 RAG 实验室、Agent 实验室和可视化实验台，把 AI 应用开发从抽象概念变成可运行、可观察、可复盘的项目。

## 2. 目标用户

第一目标用户：

- 有测试工程经验
- 会一点基础编程或愿意学 Python
- 想转向 AI 应用开发
- 希望通过真实项目学习，而不是只看教程

用户当前痛点：

- 不知道 RAG、Embedding、向量数据库到底怎么串起来
- 工具太多，不知道先学哪个
- Agent 概念很多，但不知道如何落地
- 学了很多理论，没有可以展示的项目
- 不知道 AI 系统怎么测试和评估

## 3. 产品目标

### 3.1 学习目标

帮助用户掌握：

- 大模型 API 调用
- RAG 知识库问答
- 文档解析和切片
- Embedding 和向量检索
- Hybrid 检索和 Rerank
- RAG 自动评测
- Agent 工具调用
- Agent 状态、记忆、反思和停止条件
- FastAPI 后端开发
- 数据库存储
- 本地模型和云模型接入
- 项目部署

### 3.2 项目目标

最终做成一个可以演示的 AI 应用系统：

```text
上传资料
  -> 文档解析
  -> 文本切片
  -> Embedding
  -> 向量库
  -> RAG 问答
  -> 质量评测
  -> Agent 调用工具
  -> 自动生成任务
  -> 学习和项目复盘
```

## 4. 核心模块

第一版只保留必要模块：

1. 首页
2. 学习路线
3. RAG 实验室
4. Agent 实验室
5. 可视化实验台
6. 模型与 API 配置
7. 技术栈地图
8. 任务中心

其中主角是：

- RAG 实验室
- Agent 实验室
- 可视化实验台

任务中心只作为辅助模块，用来承接学习任务和 Agent 生成的行动项，不做复杂项目管理。

暂不做：

- 多用户登录
- 权限系统
- 花哨大屏
- 复杂团队协作
- Graph RAG 完整实现
- 多租户
- 付费系统

## 5. 核心原则

### 5.1 学习优先

每个功能都要能回答三个问题：

- 这个技术是什么？
- 它在系统里怎么工作？
- 我怎么知道它做对了？

### 5.2 可观察

RAG 和 Agent 不能只给最终答案。

必须展示：

- 输入
- 中间步骤
- 工具调用
- 检索结果
- 分数
- 引用来源
- 判断原因
- 失败原因

### 5.3 先简单后复杂

学习顺序：

```text
Naive RAG
  -> Advanced RAG
  -> Agentic RAG
  -> Graph RAG
```

开发顺序：

```text
静态前端原型
  -> FastAPI 后端
  -> SQLite 存储
  -> 文档上传解析
  -> FAISS/Chroma 检索
  -> 大模型回答
  -> 评测
  -> Agent
  -> 部署
```

## 6. 第一版 MVP

第一版 MVP 必须完成：

- 清晰的首页和学习路线
- RAG 实验室基础流程演示
- 文档切片可视化
- 检索结果可视化
- Agent 实验室流程演示
- 技术栈地图
- 任务中心

第一版可以暂时不接真实后端，但页面结构必须为后续接后端预留。

## 7. 第二版 MVP

第二版开始接真实后端：

- FastAPI
- SQLite
- 文档上传
- Markdown / TXT / PDF / Word 解析
- 切片入库
- 本地 Embedding
- FAISS 或 Chroma
- RAG 问答接口
- RAG 实验记录

## 8. 第三版 MVP

第三版做 Advanced RAG：

- BM25
- Hybrid 检索
- Rerank
- 无依据拒答
- 自动评测集
- 质量报告

## 9. 第四版 MVP

第四版做 Agent：

- 工具调用
- Agent 状态流
- 运行轨迹
- 任务生成
- Agent 评测
- 可选 LangGraph
