# 迭代路线图

## V0：需求和前端原型

目标：

- 明确产品方向
- 明确模块边界
- 做出可看的前端原型

范围：

- 首页
- 学习路线
- RAG 实验室静态演示
- Agent 实验室静态演示
- 数据入库和检索命中可视化
- 技术栈地图
- 任务中心

验收：

- 用户能看懂系统要做什么
- 用户能判断页面结构是否舒服
- 不接后端也能演示核心流程
- 用户能直观看到 documents、chunks、vectors、向量索引之间的关系

## V1：RAG 基础后端

目标：

- 跑通 Naive RAG

技术：

- Python
- FastAPI
- SQLite
- 文档解析
- 文本切片
- Embedding
- FAISS 或 Chroma
- 大模型 API

功能：

- 创建知识库
- 上传文档
- 解析文本
- 切片入库
- 生成向量
- 检索 Top K
- 基于引用回答
- 可视化展示数据存储和检索命中

验收：

- 能上传文档
- 能提问
- 能看到引用
- 能看到切片和向量信息
- 能看到问题命中了哪些 chunk

## V2：Advanced RAG

目标：

- 让 RAG 从 Demo 走向可用

技术：

- BM25
- Hybrid 检索
- Rerank
- 自动评测

功能：

- BM25 检索
- 向量 + BM25 融合
- Rerank 重排序
- 无依据拒答
- 评测用例
- 批量评测
- 参数对比

验收：

- 有依据问题能答
- 无依据问题能拒答
- 评测集能跑
- 改参数能看出效果变化

## V3：Agent 基础

目标：

- 让 AI 能调用系统工具

技术：

- Tool Calling 思想
- 手写 Agent 流程
- SQLite 运行日志

功能：

- 输入目标
- 计划步骤
- 调用 RAG 检索
- 评估证据
- 生成任务
- 保存运行轨迹

验收：

- 能看到每一步工具调用
- 能保存 Agent 运行记录
- 能生成任务中心任务

## V4：真实模型 Agent

目标：

- 接真实大模型的工具调用能力

技术：

- DeepSeek / 通义千问 / OpenAI
- Tool schema
- JSON 参数校验
- 最大步数限制
- 错误重试

功能：

- 模型选择工具
- 工具参数由模型生成
- 工具调用失败可恢复
- Agent 成本和耗时统计

验收：

- Agent 可以动态选择工具
- Agent 不会无限循环
- Agent 失败原因可解释

## V5：框架化 Agent

目标：

- 学习工业级 Agent 编排

技术：

- LangGraph
- LangChain
- LlamaIndex

功能：

- 状态图
- 节点可视化
- 条件边
- Memory
- 多工具工作流

验收：

- 能解释 LangGraph 解决什么问题
- 能把手写流程迁移成状态图

## V6：部署与作品化

目标：

- 做成可以展示的完整项目

技术：

- Docker
- Nginx
- 云服务器
- PostgreSQL
- 日志监控

功能：

- 一键启动
- 配置管理
- 数据备份
- 访问地址

验收：

- 可以在另一台电脑访问
- 可以作为作品展示
