# 技术栈使用地图

## 1. 编程与环境工具

### VS Code

用途：

- 写前端
- 写 Python 后端
- 看项目结构
- 调试代码

阶段：

- 从 V0 开始一直使用

### Python

用途：

- FastAPI 后端
- 文档解析
- Embedding
- RAG 检索
- Agent 工具

阶段：

- V1 开始核心使用

### Anaconda

用途：

- 管理 Python 环境
- 隔离依赖

阶段：

- V1 使用

### Git

用途：

- 代码管理
- 版本回滚
- 上传 GitHub

阶段：

- 从 V0 开始使用

### Postman

用途：

- 测试 FastAPI 接口
- 调试模型 API

阶段：

- V1 开始使用

## 2. 大模型与 API

### DeepSeek / 通义千问 / OpenAI / 豆包

用途：

- RAG 最终回答
- Agent 推理
- 工具调用
- 总结和任务生成

阶段：

- V1 可以先接一个
- V4 做多模型配置

## 3. 向量数据库

### FAISS

用途：

- 本地向量检索
- 学习向量相似度

阶段：

- V1 推荐先用

### Chroma

用途：

- 更像完整向量数据库
- 方便本地开发

阶段：

- V1 或 V2 可替换 FAISS 做对比

### Milvus

用途：

- 企业级向量数据库
- 大规模向量检索

阶段：

- V6 或后期专项

## 4. 核心框架

### FastAPI

用途：

- 后端接口
- 文件上传
- RAG 问答 API
- Agent 运行 API

阶段：

- V1 开始使用

### LangChain

用途：

- LLM 应用编排
- Tool Calling
- Agent 工具封装

阶段：

- V4 或 V5 使用

### LlamaIndex

用途：

- 数据连接器
- RAG 索引
- 文档问答

阶段：

- V5 做对比学习

### LangGraph

用途：

- Agent 状态图
- 多步骤流程
- 条件边

阶段：

- V5 使用

## 5. 本地模型

### Ollama

用途：

- 本地跑 Qwen、Llama 等模型
- 离线实验
- 对比云模型

阶段：

- V2 后可以加入

### Qwen / Llama

用途：

- 本地问答
- 本地 Agent 实验

阶段：

- V2 或 V4

## 6. 文档处理

### pdfplumber / PyPDF2

用途：

- 解析 PDF

阶段：

- V1

### python-docx

用途：

- 解析 Word

阶段：

- V1

### OCR

用途：

- 处理扫描 PDF
- 处理文档图片里的文字

阶段：

- 后期专项

## 7. 部署

### Docker

用途：

- 打包后端和依赖
- 一键部署

阶段：

- V6

### 云服务器

用途：

- 对外访问
- 项目展示

阶段：

- V6

## 8. 测试工程师优势

你原本的测试能力会用于：

- 写 RAG 评测集
- 设计无依据问题
- 检查模型幻觉
- 做 Prompt 回归
- 做 Agent 工具调用测试
- 生成质量报告

这会成为你转 AI 应用工程师的差异化优势。
