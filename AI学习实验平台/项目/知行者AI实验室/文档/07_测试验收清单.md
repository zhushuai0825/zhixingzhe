# 测试验收清单

这份文档用于每次修改后自测。

## 1. 基础服务验收

- [ ] Docker Desktop 已启动。
- [ ] PostgreSQL 容器运行。
- [ ] Chroma 容器运行。
- [ ] 后端 `http://127.0.0.1:8010/docs` 能打开。
- [ ] 前端 `http://127.0.0.1:8088` 能打开。
- [ ] `/health` 返回 `ok=true`。

```bash
curl http://127.0.0.1:8010/health
```

## 2. 前端页面验收

- [ ] 项目总览能打开。
- [ ] RAG 实验室能打开。
- [ ] 可视化实验台能打开。

Agent 最终回答接口验证：

```bash
python3 - <<'PY'
import json, urllib.request

payload = json.dumps({
    "knowledge_base_id": "",
    "agent_type": "learning",
    "goal": "Agent 是什么？",
    "enable_reflect": True,
    "top_k": 3
}, ensure_ascii=False).encode("utf-8")

req = urllib.request.Request(
    "http://127.0.0.1:8010/api/agent-runs",
    data=payload,
    method="POST",
    headers={"Content-Type": "application/json"}
)

with urllib.request.urlopen(req, timeout=60) as resp:
    data = json.loads(resp.read().decode("utf-8"))

print("has_final_answer:", bool(data.get("final_answer")))
print("citation_count:", len(data.get("citations", [])))
print("first_citation_has_content:", bool(data.get("citations", [{}])[0].get("content")))
PY
```

## 9. 回归重点

以后改代码最容易影响这些地方：

- 删除知识库后，Agent 是否还拿旧知识库 ID。
- 修改 chunk 结构后，Chroma metadata 是否同步。
- 修改 Agent 返回结构后，前端最终结果是否还能展示。
- 修改检索分数后，证据门槛是否过松或过严。
- 修改前端布局后，移动端是否还能看。
