# 后续迭代计划

当前版本已经完成学习型 RAG + Agent 闭环。后续不要一下子做太散，建议按下面顺序推进。

## 1. V1 当前稳定版

当前已经完成：

- RAG 入库。
- 文档切片。
- PostgreSQL 存储。
- Chroma 向量库。
- 本地学习版 Embedding。
- BM25。
- Hybrid 检索。
- 大模型最终回答。
- 引用来源展开。
- Agent 运行轨迹。
- Agent 任务生成。
- 数据中心。
- 3D 向量空间。

目标：先把当前版本用熟。

## 2. V1.1 真实 Embedding

学习目标：

- 真实 Embedding 和哈希向量的差异。
- 向量维度。
- 中文语义检索。
- 批量向量化。
- 向量重建。

可选方案：

- bge-small-zh 本地模型。
- bge-m3。
- 通义 Embedding API。
- OpenAI 兼容 Embedding API。

验收标准：

- 可以切换 Embedding provider。
- 新旧向量不能混用。
- 数据中心显示 embedding provider。
- 可视化实验台能比较检索差异。

## 3. V1.2 Rerank 重排序

学习目标：

- 召回和排序的区别。
- 为什么 Top K 初筛后还要 rerank。
- Cross Encoder / LLM rerank。

验收标准：

- 检索结果展示 recall_score 和 rerank_score。
- 可以开关 rerank。
- Agent 使用 rerank 后的证据。

## 4. V1.3 RAG 自动评测

学习目标：

- 什么是评测集。
- Recall@K。
- Precision@K。
- Faithfulness。
- 引用正确率。
- 拒答率。

验收标准：

- 可以录入一批问题。
- 批量运行 RAG。
- 输出质量报告。
- 标记失败样例。

## 5. V1.4 文档处理增强

学习目标：

- Word 解析。
- OCR。
- 图片提取。
- 表格解析。
- 多模态 RAG 入门。

验收标准：

- 支持 `.docx`。
- 支持扫描 PDF OCR。
- 支持图片提取文字。
- 数据中心标记解析方式。

## 6. V2 Agent 工具调用

学习目标：

- Tool Calling。
- JSON schema。
- 工具参数校验。
- 工具调用失败处理。
- 最大步数和停止条件。

建议工具：

- `search_knowledge_base`
- `list_documents`
- `get_chunk_detail`
- `create_task`
- `evaluate_answer`

验收标准：

- 大模型能选择工具。
- 前端能看到每次工具调用。
- 工具输入输出结构化展示。
- Agent 不会无限循环。

## 7. V2.1 LangGraph

学习目标：

- 状态图。
- 节点。
- 边。
- 条件跳转。
- checkpoint。

验收标准：

- 当前手写 Agent 流程迁移成 LangGraph。
- 页面可视化显示状态图。
- 每个节点保存状态快照。

## 8. V2.2 Agent 评测

学习目标：

- Agent 是否完成目标。
- 工具调用是否合理。
- 是否过度检索。
- 是否引用正确。
- 是否能失败停止。

验收标准：

- 给 Agent 一组任务。
- 自动运行。
- 输出成功率、失败原因、平均工具调用次数。

## 9. V3 Graph RAG

学习目标：

- 实体抽取。
- 关系抽取。
- 知识图谱。
- 多跳推理。
- 图数据库。

可选技术：

- Neo4j。
- NetworkX。
- PostgreSQL 表结构模拟图。

验收标准：

- 文档能抽取实体和关系。
- 页面显示图谱。
- 问题能命中实体和多跳关系。

## 10. V4 部署和产品化

学习目标：

- Docker 打包。
- Nginx。
- 云服务器。
- 日志。
- 备份。
- 权限。

验收标准：

- 一条命令启动完整系统。
- 环境变量配置模型。
- 数据可备份。
- 有基本登录保护。

## 11. 学习优先级

建议你接下来按这个顺序学：

1. 把当前系统每个页面用熟。
2. 学真实 Embedding。
3. 学 Rerank。
4. 学 RAG 评测。
5. 学 Tool Calling。
6. 学 LangGraph。
7. 学 Graph RAG。
8. 学部署。
