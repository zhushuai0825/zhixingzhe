# 数据与存储说明

这份文档解释系统里的数据怎么存。

## 1. 总体关系

当前系统使用两个核心存储：

```text
PostgreSQL：业务数据库
Chroma：向量数据库
```

二者不是互相替代关系，而是分工合作。

## 2. PostgreSQL 存什么

PostgreSQL 存业务数据。

当前表：

- `knowledge_bases`：知识库。
- `documents`：文档。
- `document_chunks`：文档切片。
- `rag_runs`：RAG 运行记录。
- `agent_runs`：Agent 运行记录。
- `tasks`：任务。

## 3. Chroma 存什么

Chroma 存向量数据。

当前 collection：

```text
document_vectors
```

每个向量包含：

- id：chunk_id。
- embedding：chunk 向量。
- document：chunk 原文。
- metadata：knowledge_base_id、document_id、chunk_index、file_name。

## 4. 为什么不只用 PostgreSQL

普通 PostgreSQL 擅长：

- 精确查询。
- 关系数据。
- 事务。
- 业务记录。

向量数据库擅长：

- 高维向量索引。
- 相似度检索。
- Top K 查询。

所以当前系统选择：

```text
PostgreSQL 保存“是什么”
Chroma 负责“像不像”
```

## 5. 文档入库数据流

```text
用户上传文档
  -> 后端解析文本
  -> chunk_text 切片
  -> documents 写入 PostgreSQL
  -> document_chunks 写入 PostgreSQL
  -> embed_text 生成向量
  -> chroma_upsert 写入 Chroma
```

## 6. 检索数据流

```text
用户输入问题
  -> embed_text 生成问题向量
  -> Chroma 查询相似向量
  -> PostgreSQL 读取所有 chunk 做 BM25
  -> 合并 vector_score 和 keyword_score
  -> diversify_hits 去重
  -> build_grounded_answer 调用大模型
  -> rag_runs 保存运行记录
```

## 7. Agent 数据流

```text
用户输入目标
  -> run_agent
  -> 调用 search_knowledge_base
  -> 得到 answer、hits、retrieval_trace
  -> 构建 Agent steps
  -> 构建 tasks
  -> agent_runs 保存运行轨迹
  -> 前端展示 final_answer、citations、tasks、steps
```

## 8. 删除数据

删除知识库：

- 删除 PostgreSQL 中的知识库、文档、切片和运行记录。
- 删除 Chroma 中对应 chunk 向量。

删除文档：

- 删除 PostgreSQL 中的文档和切片。
- 删除 Chroma 中对应 chunk 向量。

## 9. 重要字段说明

### knowledge_bases

| 字段 | 说明 |
| --- | --- |
| id | 知识库 ID |
| name | 知识库名称 |
| description | 描述 |
| created_at | 创建时间 |

### documents

| 字段 | 说明 |
| --- | --- |
| id | 文档 ID |
| knowledge_base_id | 所属知识库 |
| file_name | 文件名 |
| file_type | 文件类型 |
| file_size | 文件大小 |
| status | 状态 |
| summary | 简要说明 |

### document_chunks

| 字段 | 说明 |
| --- | --- |
| id | chunk ID |
| document_id | 所属文档 |
| knowledge_base_id | 所属知识库 |
| chunk_index | 文档内序号 |
| content | 原文片段 |
| token_count | 当前用字符长度近似 |

### rag_runs

| 字段 | 说明 |
| --- | --- |
| id | RAG 运行 ID |
| question | 用户问题 |
| answer | 最终回答 |
| citations_json | 引用信息 |
| evaluation_json | 检索解释和评估信息 |

### agent_runs

| 字段 | 说明 |
| --- | --- |
| id | Agent 运行 ID |
| goal | 用户目标 |
| mode | Agent 类型 |
| summary | 总结 |
| steps_json | Agent 步骤轨迹 |

## 10. 数据检查命令

进入 PostgreSQL：

```bash
docker exec -it zhixingzhe-ai-postgres psql -U zhixingzhe -d zhixingzhe_ai_lab
```

查看知识库：

```sql
SELECT id, name, created_at FROM knowledge_bases ORDER BY created_at DESC;
```

查看文档：

```sql
SELECT id, file_name, file_type, status FROM documents ORDER BY created_at DESC;
```

查看切片数量：

```sql
SELECT knowledge_base_id, count(*) FROM document_chunks GROUP BY knowledge_base_id;
```

查看最近 RAG 记录：

```sql
SELECT id, question, created_at FROM rag_runs ORDER BY created_at DESC LIMIT 5;
```

查看最近 Agent 记录：

```sql
SELECT id, goal, summary, created_at FROM agent_runs ORDER BY created_at DESC LIMIT 5;
```
