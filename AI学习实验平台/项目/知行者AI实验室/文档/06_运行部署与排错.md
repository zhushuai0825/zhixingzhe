# 运行部署与排错

这份文档用于以后启动、停止、检查和排查知行者 AI 实验室。

## 1. 一键检查顺序

每次系统不正常，先按这个顺序查：

1. Docker Desktop 是否启动。
2. PostgreSQL 和 Chroma 容器是否启动。
3. 后端 FastAPI 是否启动。
4. 前端静态服务是否启动。
5. `/health` 是否正常。
6. 浏览器是否强刷新。

## 2. 启动基础设施

```bash
cd /Users/zhushuai/Downloads/知行者AI实验室
docker compose up -d
```

查看状态：

```bash
docker compose ps
```

## 3. 启动后端

```bash
cd /Users/zhushuai/Downloads/知行者AI实验室/后端
source .venv/bin/activate
uvicorn app.main:app --host 127.0.0.1 --port 8010
```

## 4. 启动前端

```bash
cd /Users/zhushuai/Downloads/知行者AI实验室/前端原型
python3 -m http.server 8088
```

## 5. 检查服务

检查后端：

```bash
curl http://127.0.0.1:8010/health
```

正常返回：

```json
{
  "ok": true,
  "postgres": true,
  "chroma_collection_count": 109
}
```

检查前端：

```bash
curl http://127.0.0.1:8088/
```

检查端口进程：

```bash
pgrep -fl 'uvicorn app.main:app|http.server 8088'
```

## 6. 修改后端后怎么生效

后端 Python 改了以后，需要重启 FastAPI。

找到进程：

```bash
pgrep -fl 'uvicorn app.main:app'
```

停止进程：

```bash
kill <pid>
```

重新启动：

```bash
cd /Users/zhushuai/Downloads/知行者AI实验室/后端
source .venv/bin/activate
uvicorn app.main:app --host 127.0.0.1 --port 8010
```

## 7. 修改前端后怎么生效

前端是静态文件服务。

如果浏览器没看到新效果，按：

```text
Command + Shift + R
```

这是强刷新，会清掉浏览器缓存。

## 8. 常见问题

### 8.1 前端打不开

现象：

```text
http://127.0.0.1:8088 打不开
```

检查：

```bash
pgrep -fl 'http.server 8088'
```

如果没有进程，重新启动前端。

### 8.2 后端打不开

现象：

```text
http://127.0.0.1:8010/docs 打不开
```

检查：

```bash
pgrep -fl 'uvicorn app.main:app'
```

如果没有进程，重新启动后端。

### 8.3 PostgreSQL 失败

现象：

```json
{"postgres": false}
```

检查容器：

```bash
docker compose ps
```

查看日志：

```bash
docker logs zhixingzhe-ai-postgres
```

### 8.4 Chroma 请求失败

现象：

```text
Chroma 请求失败
```

检查：

```bash
curl http://127.0.0.1:8001/api/v2/heartbeat
docker logs zhixingzhe-ai-chroma
```

如果 Chroma 容器是 healthy，但后端 `/health` 仍提示 Chroma 连接失败，优先检查 `.env`：

```text
CHROMA_HOST=127.0.0.1
CHROMA_PORT=8001
```

本项目推荐使用 `127.0.0.1`，避免部分环境中 `localhost` 解析到 IPv6 后连接失败。

### 8.5 大模型没有回答

可能原因：

- `.env` 没有配置 API Key。
- base_url 不正确。
- 模型名不正确。
- 网络不可用。
- 检索分数低于 min_score，系统拒答。

检查 `.env`：

```text
SENSEAUDIO_BASE_URL=...
SENSEAUDIO_API_KEY=...
SENSEAUDIO_CHAT_MODEL=deepseek-v4-pro
```

不要把真实 API Key 写进公开文档或提交到 GitHub。

### 8.6 上传 PDF 后没有内容

可能原因：

- PDF 是扫描件。
- PDF 是图片，不是真文字。
- 当前系统暂未接 OCR。

解决方向：

- 换可复制文字的 PDF。
- 先转成 txt/md。
- 后续接 OCR。

### 8.7 引用来源看不到原文

处理：

- 强刷新页面。
- 重新运行 Agent。
- 确认后端已重启。

当前版本引用来源应该可以展开看到原文。

## 9. 保留数据和清空数据

停止容器但保留数据：

```bash
docker compose stop
```

删除容器但保留 volume：

```bash
docker compose down
```

如果未来要彻底清空数据，需要额外删除 Docker volume。这个操作会删除所有知识库和向量，谨慎执行。
