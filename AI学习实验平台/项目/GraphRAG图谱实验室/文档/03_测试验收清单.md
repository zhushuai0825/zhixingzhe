# Graph RAG 测试验收清单

## 1. 页面检查

- 页面能从平台左上角切换进入。
- 项目总览、图谱实验台、3D 图谱空间、学习路线、面试题都能打开。
- 手机宽度下不出现明显横向溢出。

## 2. 图谱实验台

- 修改问题后点击运行，路径结果会变化。
- 修改最大跳数后，路径数量或路径长度会变化。
- 新增实体后，图谱数据数量会增加。
- 新增关系后，路径检索可以使用这条关系。
- 新增证据片段后，证据与回答草稿会使用新片段。
- 重置演示数据后恢复默认实体、关系、证据。

## 3. 3D 图谱空间

- 画布不空白。
- 节点和关系线能显示。
- 鼠标拖拽能旋转。
- 滚轮能缩放。
- 悬停节点能看到完整文字说明。
- 点击节点后右侧当前节点解释会变化。

## 5. 学习效果检查

学完本模块后，应该能回答：

- Graph RAG 和普通向量 RAG 有什么区别。
- 为什么 Graph RAG 不是替代向量 RAG，而是补充。
- 实体、关系、三元组、多跳路径分别是什么。
- 为什么图谱路径不能直接当最终答案。
- 为什么最终仍然要引用原文 chunk。
- 什么时候适合用 Neo4j，什么时候 PostgreSQL 表也够用。
- Graph RAG 怎么和 Hybrid 检索、Rerank、Agent 组合。
