# Graph RAG 图谱实验室技术架构规划

## 1. 当前架构

```text
index.html
  -> 页面结构
styles.css
  -> 页面布局和响应式
app.js
  -> 数据、图遍历、证据召回、页面渲染
graph-space.js
  -> Three.js 3D 图谱可视化
vendor/three.module.js
  -> 本地 Three.js 依赖
```

## 2. 数据存储

当前使用浏览器 `localStorage` 保存用户新增实体、关系和证据。

这样做的好处是：

- 不需要后端就能学习。
- 刷新后数据仍在。
- 适合快速试验实体关系变化。

## 3. 后续真实工程化

下一阶段可以升级为：

```text
FastAPI
  -> /entities
  -> /relations
  -> /chunks
  -> /graph/search

PostgreSQL
  -> graph_entities
  -> graph_relations
  -> graph_chunks

Neo4j 或 NebulaGraph
  -> 节点
  -> 边
  -> 路径查询
```

## 4. 和其他模块关系

- 文本切分实验室：负责 chunk 怎么产生。
- Embedding 实验室：负责语义向量怎么产生。
- 向量库实验室：负责 chunk embedding 怎么查询。
- Rerank 实验室：负责多路候选怎么提准。
- RAG 评测实验室：负责判断 Graph RAG 是否真的更好。
- 知行者 AI 实验室：后续可以把 Graph RAG 做成系统能力。
