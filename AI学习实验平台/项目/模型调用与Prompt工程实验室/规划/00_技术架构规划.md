# 模型调用与 Prompt 工程实验室技术架构规划

## 1. 当前架构

```text
index.html
styles.css
app.js
```

当前为纯前端教学模拟器。

## 2. 后续真实工程

后端建议：

```text
FastAPI
  -> model client
  -> prompt template manager
  -> structured output validator
  -> retry middleware
  -> usage logger
```

数据库建议：

```text
model_requests
prompt_versions
model_usage_daily
model_errors
```

## 3. 安全要求

- API Key 只放后端环境变量。
- 前端不能出现真实 Key。
- 日志要脱敏。
- 用户输入要做长度限制。
- 错误返回不要暴露供应商密钥或内部堆栈。
