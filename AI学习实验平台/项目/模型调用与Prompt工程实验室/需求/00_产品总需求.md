# 模型调用与 Prompt 工程实验室产品需求

## 1. 背景

用户已经学习 RAG、Graph RAG 和 Agent，但真实系统最终都要调用大模型 API。需要单独学习模型调用工程。

## 2. 产品目标

让用户理解：

- Prompt 怎么组织。
- 上下文怎么拼接。
- JSON Schema 怎么约束输出。
- 输出怎么校验。
- 失败怎么重试。
- Token、成本、延迟怎么估算。

## 3. 核心功能

- 编辑 system prompt。
- 编辑用户问题。
- 编辑上下文。
- 配置 schema 字段。
- 切换模型策略。
- 调整 temperature 和 max output。
- 开关 JSON、引用校验、重试。
- 展示调用过程、输出、成本、诊断和链路图。
