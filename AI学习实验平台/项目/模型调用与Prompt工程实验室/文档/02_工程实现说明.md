# 模型调用与 Prompt 工程实现说明

## 1. 当前版本定位

当前是纯前端教学模拟器，不调用真实模型 API。

它模拟：

- 消息拼接。
- Token 估算。
- 模型策略选择。
- 输出生成。
- JSON 校验。
- 引用校验。
- 自动重试。
- 成本和延迟估算。

## 2. 核心数据

### 2.1 模型配置

```js
{ name, input, output, latency, quality }
```

### 2.2 调用步骤

```js
{ type, title, detail, ok }
```

### 2.3 模拟结果

```js
{
  inputTokens,
  outputTokens,
  inputCost,
  outputCost,
  totalCost,
  latency,
  validation,
  output,
  diagnosis
}
```

## 3. 后续真实后端接口

可以设计：

```text
POST /api/model/chat
POST /api/model/structured
GET  /api/model/usage
GET  /api/model/logs/{request_id}
```

请求示例：

```json
{
  "model": "deepseek-v4-pro",
  "messages": [
    { "role": "system", "content": "只基于上下文回答" },
    { "role": "user", "content": "Graph RAG 是什么" }
  ],
  "temperature": 0.2,
  "response_format": {
    "type": "json_schema",
    "schema": {
      "answer": "string",
      "citations": "array"
    }
  }
}
```

## 4. 工程风险

- API Key 泄露。
- 请求超时。
- 供应商限流。
- 输出格式不稳定。
- JSON 解析失败。
- 成本失控。
- 日志记录了敏感信息。
- 不同供应商接口细节不同。

## 5. 和其他模块关系

- RAG：把检索证据放进上下文。
- Rerank：减少无关上下文，降低成本和幻觉。
- Graph RAG：提供路径证据。
- Agent：把模型调用作为计划、工具选择、结果总结的一部分。
