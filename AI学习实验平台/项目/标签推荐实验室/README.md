# 标签推荐实验室

标签推荐是推荐系统里最适合入门的一类方法：它不先看“谁和谁一起喜欢过什么”，而是先看内容本身有什么标签，再把用户喜欢过的内容标签累加成画像。

核心流程：

```text
物品标签
  -> 用户喜欢过的物品
  -> 累加成用户画像标签权重
  -> 候选物品标签匹配画像
  -> 过滤已喜欢物品
  -> Top N 推荐
  -> 3D 标签匹配可视化
```

## 学什么

- 标签从哪里来：人工运营、分类体系、NLP 抽取、Embedding 聚类、图片识别。
- 用户画像怎么来：把用户历史物品的标签按次数或权重累加。
- 推荐分怎么来：候选标签命中画像标签越多，分数越高。
- 为什么适合冷启动：新物品只要有标签，即使没有行为也能参与推荐。
- 常见问题：标签质量差、兴趣越推越窄、多样性不足。

## 怎么自己测试

页面里有 `实验数据中心`：

1. 先新增一个物品，填标题和标签。
2. 再给当前用户添加喜欢行为。
3. 观察用户画像标签权重是否变化。
4. 观察推荐结果和 3D 可视化连线是否变化。

这个实验室和 UserCF、SVD、推荐系统总览共用同一份本地数据，所以你在这里新增的数据，切到其他推荐实验室也可以继续用。
