# 标签推荐实验室文档索引

## 1. 一句话理解

标签推荐就是：用户过去喜欢过哪些标签，系统就优先推荐标签相近的新内容。

## 2. 数据结构

```text
item = {
  id: "i1",
  title: "RAG 入门",
  tags: ["AI", "RAG", "检索"]
}

interaction = ["u1", "i1"]
```

用户行为告诉系统“这个用户喜欢过哪些物品”，物品标签告诉系统“这个物品属于哪些方向”。

## 3. 计算过程

1. 找到当前用户喜欢过的物品。
2. 读取这些物品的标签。
3. 统计标签出现次数，得到用户画像。
4. 找出用户没喜欢过的候选物品。
5. 候选标签命中画像标签就加分。
6. 按分数排序输出 Top N。

教学版公式：

```text
候选得分 = Σ 用户画像中命中的标签权重 / sqrt(候选标签数量)
```

## 4. 为什么要除以 sqrt(候选标签数量)

如果一个物品标签特别多，它更容易“碰巧命中”用户画像。除以标签数量的平方根，是为了减少标签过多带来的虚高分。

## 5. 3D 可视化怎么看

- 中心节点：当前用户画像。
- 周围节点：候选物品。
- 连线越粗：标签命中越强。
- 节点越靠近中心：匹配程度越高。
- 悬停节点：看命中标签、物品标签和推荐分。

## 6. 面试题

### 标签推荐和协同过滤有什么区别？

标签推荐看内容标签，协同过滤看用户行为。标签推荐更适合冷启动，新物品只要有标签就能推荐；协同过滤更依赖大量用户行为。

### 标签推荐有什么缺点？

第一，标签质量决定推荐质量；第二，标签太粗会不准，太细会召回少；第三，只按标签推荐容易让用户兴趣变窄，需要和热门、协同过滤、Embedding 推荐混合。

### 标签从哪里来？

可以人工维护，也可以从标题、正文、类目、关键词抽取、模型分类、Embedding 聚类、图片识别里生成。工业系统通常会同时使用人工标签和算法标签。

### 冷启动时为什么常用标签推荐？

因为新物品还没有点击、收藏、购买行为，ItemCF 和 UserCF 很难计算相似度。但只要新物品有标签，就可以和用户画像匹配。

## 7. 从 0 到 1 手算一遍

假设当前用户喜欢过 3 个内容：

```text
RAG 入门：AI、RAG、检索
Embedding 与相似度：AI、向量、相似度
向量数据库实战：AI、向量、数据库
```

那么用户画像就是：

```text
AI = 3
向量 = 2
RAG = 1
检索 = 1
相似度 = 1
数据库 = 1
```

候选 `Rerank 精排` 的标签是：

```text
AI、排序、RAG
```

它命中了 `AI=3` 和 `RAG=1`，所以基础命中分是：

```text
3 + 1 = 4
```

再除以 `sqrt(3)`，是为了避免标签很多的物品天然占便宜：

```text
4 / sqrt(3) = 2.3094
```

所以它会排得比较靠前。

## 8. 工程落地怎么做

真实系统通常会这样做：

1. 建一张物品表，保存 `item_id、title、category、tags`。
2. 建一张行为表，保存 `user_id、item_id、action、timestamp`。
3. 离线或实时聚合用户画像，例如 `user_id -> tag_weight`。
4. 查询时先过滤用户已看/已买/已喜欢物品。
5. 对候选物品按标签命中分排序。
6. 加上业务规则，例如去重、多样性、黑名单、发布时间、质量分。
7. 返回 Top N，并给出推荐解释。

## 9. 常见误区

- 误区一：标签越多越好。标签太多会制造噪声，推荐反而变散。
- 误区二：只要标签命中就一定好。还要考虑内容质量、时效、用户是否已经看过。
- 误区三：标签推荐能理解语义。它只能匹配标签，不理解两个不同标签背后是不是语义相近。
- 误区四：冷启动只靠标签就够。标签推荐是兜底能力，后面仍然要接行为推荐和排序模型。

## 10. 面试回答模板

如果面试官问“你怎么实现一个标签推荐”，可以这样回答：

```text
我会先把物品打上标签，标签可以来自人工、类目、关键词抽取或模型分类。
然后根据用户历史行为，把用户喜欢过的物品标签聚合成用户画像，画像里每个标签有权重。
推荐时先过滤用户已经看过或喜欢过的物品，再计算候选物品标签和用户画像的匹配分。
最后按分数排序取 Top N，同时加上多样性、质量分和业务规则。
它的优点是适合新物品冷启动，缺点是依赖标签质量，容易推荐范围变窄。
```

## 11. 学完要能回答

- 用户画像和物品标签分别是什么。
- 为什么候选物品要过滤已喜欢内容。
- 推荐分怎么计算。
- 为什么标签推荐适合冷启动。
- 标签推荐为什么不能替代 Embedding 推荐。
- 工业系统里为什么要和热门、ItemCF、SVD、排序模型混合。

## 12. 动手实验建议

建议你这样测：

1. 新增一个 `Graph RAG 实战课`，标签填 `AI, RAG, 图谱`。
2. 给小周添加喜欢这个物品。
3. 看用户画像里 `AI`、`RAG` 权重是否上升。
4. 再新增一个 `Agentic RAG 项目课`，标签填 `AI, RAG, Agent`。
5. 如果它没有被当前用户喜欢过，就应该更容易进入推荐结果。

这样你就能直观看到：标签推荐不是凭空生成推荐，而是把“用户历史喜欢的标签”拿来和“候选物品标签”做匹配。
