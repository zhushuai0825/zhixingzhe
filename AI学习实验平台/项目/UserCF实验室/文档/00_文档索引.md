# UserCF 实验室文档索引

## 1. 一句话理解

UserCF 就是：先找和当前用户兴趣相似的人，再推荐这些相似用户喜欢、当前用户还没喜欢过的物品。

## 2. 数据结构

```text
小周喜欢：RAG 入门、Embedding 与相似度、向量数据库实战
小林喜欢：RAG 入门、Embedding 与相似度、向量数据库实战、Rerank 精排
```

小周和小林有 3 个共同喜欢，所以他们相似。小林额外喜欢的 `Rerank 精排` 就可能推荐给小周。

## 3. 相似度公式

```text
sim(u, v) = 共同喜欢数量 / sqrt(u 喜欢数量 * v 喜欢数量)
```

这个公式的意义是：共同喜欢越多，相似度越高；但如果某个人喜欢特别多，也不能让他随便和谁都很像。

## 4. 推荐分怎么来

如果多个相似用户都喜欢同一个候选物品，这个候选物品的分数会累加。

```text
候选物品得分 = Σ 喜欢过该物品的相似用户相似度
```

## 5. 3D 可视化怎么看

- 中心或左侧：当前用户。
- 周围：其他用户。
- 越靠近：用户行为越相似。
- 右侧：相似用户贡献出来的推荐候选。
- 连线：相似用户把候选物品推荐过来的路径。

## 6. 面试题

### UserCF 和 ItemCF 有什么区别？

UserCF 问“哪个用户和我像”；ItemCF 问“我喜欢的物品和哪些物品像”。UserCF 更直观，适合理解人群兴趣；ItemCF 通常更稳定，更适合工业缓存。

### UserCF 的优点是什么？

它不需要理解物品内容，只要有用户行为就可以工作；同时解释也比较自然，可以说“因为和你相似的人喜欢了这个”。

### UserCF 的缺点是什么？

用户数量大时，相似用户计算成本高；用户兴趣变化快时，相似关系容易过期；新用户没有行为时无法找到相似用户。

### 工业里怎么优化？

通常离线计算相似用户或召回候选，线上只做快速合并、过滤、排序。还会加入热门推荐、标签推荐、地域、时间、业务规则等兜底策略。

## 7. 从 0 到 1 手算一遍

假设有两个用户：

```text
小周喜欢：RAG、Embedding、向量库、A/B 测试
小林喜欢：RAG、Embedding、向量库、Rerank、Agent
```

共同喜欢的是：

```text
RAG、Embedding、向量库
```

共同喜欢数量是 3。小周喜欢 4 个，小林喜欢 5 个。

```text
sim(小周, 小林) = 3 / sqrt(4 * 5)
                = 3 / sqrt(20)
                = 0.6708
```

小林喜欢但小周没喜欢过的是：

```text
Rerank、Agent
```

所以这两个内容会成为候选。如果另一个相似用户也喜欢 `Rerank`，那么 `Rerank` 的贡献分还会继续累加。

## 8. 工程落地怎么做

真实系统不会在用户请求时全量比较所有用户，通常这样做：

1. 收集用户行为：点击、收藏、购买、播放、评分等。
2. 清洗行为：过滤异常点击、机器人行为、太老的数据。
3. 离线构建用户-物品倒排表。
4. 离线计算用户相似度，只保留每个用户最相似的 Top M 用户。
5. 在线推荐时读取相似用户列表。
6. 合并相似用户喜欢过的物品。
7. 过滤当前用户已看、已买、不适合推荐的物品。
8. 按相似度贡献、时间、热度、质量分排序。
9. 返回 Top N，并输出“相似用户喜欢过”的解释。

## 9. 常见误区

- 误区一：相似用户越多越好。太多低相似用户会引入噪声。
- 误区二：UserCF 能理解内容。它只看行为，不理解文本或标签。
- 误区三：新用户也能直接算。新用户没有行为，很难找到相似用户。
- 误区四：线上实时全量算相似度。大规模系统通常离线预计算。
- 误区五：只看点击。点击不一定代表喜欢，最好结合停留、收藏、购买、评分等更强信号。

## 10. 面试回答模板

如果面试官问“UserCF 怎么做”，可以这样回答：

```text
UserCF 先把每个用户表示成一个行为集合，例如喜欢过哪些物品。
然后计算当前用户和其他用户的相似度，常见方法是余弦相似度或 Jaccard。
找到相似用户后，把相似用户喜欢而当前用户没喜欢过的物品作为候选。
候选物品的得分等于喜欢过它的相似用户相似度之和。
最后过滤已看、去重、排序，输出 Top N。
它的优点是解释直观，缺点是用户规模大时计算成本高，新用户冷启动困难。
```

## 11. 学完要能回答

- 用户相似度为什么用集合重合计算。
- 为什么要过滤当前用户已喜欢物品。
- 多个相似用户如何共同贡献一个推荐。
- UserCF 为什么有新用户冷启动问题。
- UserCF 和 ItemCF 的区别。
- 为什么工业系统通常离线计算相似用户。

## 12. 动手实验建议

建议你手动构造一个“相似用户”：

1. 新增用户 `小王`。
2. 给小王添加 `RAG 入门`、`Embedding 与相似度`、`向量数据库实战`。
3. 再给小王添加 `Rerank 精排`。
4. 切换到小周。
5. 观察小王和小周的相似度，以及 `Rerank 精排` 是否因为小王贡献而进入推荐。

如果你把小王的共同喜欢删掉，相似度会下降，推荐贡献也会下降。这就是 UserCF 最核心的因果关系。
