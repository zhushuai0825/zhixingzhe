# UserCF 实验室

UserCF 是基于用户的协同过滤。它的核心思路很口语化：和你兴趣相似的人喜欢什么，你可能也会喜欢什么。

核心流程：

```text
用户-物品行为
  -> 每个用户的喜欢集合
  -> 计算用户相似度
  -> 找相似用户喜欢、当前用户没喜欢的物品
  -> 累加相似用户贡献
  -> Top N 推荐
  -> 3D 用户相似可视化
```

## 学什么

- 用户行为集合如何表示。
- 用户相似度如何计算。
- 推荐候选如何从相似用户那里产生。
- 为什么要过滤已喜欢物品。
- UserCF 和 ItemCF 的区别。
- 大规模系统为什么更常用离线预计算。

## 怎么自己测试

页面里的 `实验数据中心` 可以新增用户、物品和喜欢行为。

推荐你这样试：

1. 新增一个用户 `小王`。
2. 给小王添加几条和小周相同的喜欢。
3. 再给小王添加一条小周没有喜欢过的内容。
4. 切回小周，看 UserCF 是否把小王额外喜欢的内容推荐出来。

UserCF 的关键就是“相似用户贡献候选”，所以你越能手动制造共同喜欢和差异喜欢，就越容易看懂它。
