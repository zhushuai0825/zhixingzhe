# Agent 工程实验室产品需求

## 1. 背景

用户已经学习 RAG、Rerank、RAG 评测和 Graph RAG。下一步需要学习如何把这些能力组织成可控的工具调用系统。

## 2. 产品目标

做一个可交互 Agent 学习实验室，让用户直观看到：

- 目标如何变成计划。
- 计划如何选择工具。
- 工具输入输出是什么。
- 证据如何被观察和评估。
- 证据不足时为什么不能硬答。
- 反思检查如何给出下一步最小行动。

## 3. 核心功能

- 输入目标。
- 切换 Agent 类型。
- 开关 RAG、Graph RAG、反思工具。
- 调整最大工具调用步数。
- 调整证据阈值。
- 新增知识片段。
- 新增工具。
- 查看执行轨迹。
- 查看最终结果。
- 查看轨迹图和面试题。

## 4. 成功标准

- 输入变了，结果会变。
- 工具开关变了，轨迹会变。
- 用户能看到每一步的输入和输出。
- 页面能解释 Agent 的工程边界。
