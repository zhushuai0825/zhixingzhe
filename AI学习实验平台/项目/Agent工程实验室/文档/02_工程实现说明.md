# Agent 工程实现说明

## 1. 当前版本定位

当前是纯前端教学版，用固定规则模拟 Agent 执行。

它不调用真实大模型，但保留真实 Agent 工程里的核心概念：

- 目标。
- 计划。
- 工具调用。
- 观察。
- 证据门。
- 任务生成。
- 反思检查。
- 执行轨迹。

## 2. 数据结构

### 2.1 知识片段

```js
{ id, title, tags, text }
```

### 2.2 工具

```js
{ id, name, desc, enabled }
```

### 2.3 执行步骤

```js
{ id, type, name, title, input, output, explain, score }
```

## 3. 执行流程

1. 读取用户目标。
2. 根据 Agent 类型生成计划。
3. 如果启用 RAG，调用本地知识检索。
4. 如果目标涉及图谱、多跳、关系，调用 Graph RAG 模拟工具。
5. 合并证据。
6. 通过证据门判断是否足够。
7. 生成任务。
8. 如果启用反思，检查下一步最小行动和风险。
9. 渲染最终结果和轨迹图。

## 5. 后续升级到 LangGraph

LangGraph 版本可以设计这些节点：

```text
planner
  -> rag_search
  -> graph_search
  -> evidence_gate
  -> task_writer
  -> reflection_check
  -> final_answer
```

状态对象可以是：

```python
class AgentState(TypedDict):
    goal: str
    plan: list[str]
    evidence: list[dict]
    steps: list[dict]
    final_answer: str
    error: str | None
```

## 6. 工程风险

- 工具参数错误。
- 工具输出为空。
- 模型误读工具输出。
- 循环调用工具。
- 证据不足还硬生成。
- 没有日志，无法排查每一步为什么这么做。
