# Agent 零基础学习手册

## 1. Agent 是什么

你可以先这样理解：

```text
普通问答：用户问 -> 模型答
Agent：用户给目标 -> 模型计划 -> 调工具 -> 看结果 -> 再决定下一步 -> 输出结果
```

Agent 不是“模型突然有自主意识”，而是一个工程系统。

它通常包含：

- 目标。
- 状态。
- 工具。
- 计划器。
- 执行器。
- 观察器。
- 评估器。
- 反思器。
- 停止条件。

## 2. 为什么要学 Agent

RAG 解决的是“基于知识回答问题”。

Agent 解决的是“围绕目标完成一串动作”。

例子：

```text
目标：我想学习 Graph RAG，下一步做什么？
```

普通问答可能直接给建议。

Agent 应该：

1. 判断目标类型是学习规划。
2. 检索 Graph RAG 资料。
3. 如有必要调用图谱检索。
4. 判断证据是否够。
5. 生成学习任务。
6. 检查任务是否太大。
7. 输出下一步行动。

## 3. Tool Calling 是什么

Tool Calling 就是让模型按固定格式调用函数。

比如你定义一个工具：

```json
{
  "name": "rag_search",
  "description": "从知识库检索资料",
  "parameters": {
    "query": "用户问题",
    "top_k": 5
  }
}
```

模型不能真的自己查数据库，它只是输出：

```json
{
  "tool": "rag_search",
  "arguments": {
    "query": "Graph RAG 是什么",
    "top_k": 5
  }
}
```

真正执行查询的是你的后端代码。

## 4. Agent 和 Tool Calling 的区别

Tool Calling 是一次工具调用能力。

Agent 是多步流程控制。

```text
Tool Calling：调用一个工具
Agent：决定是否调用工具、调用哪个工具、怎么处理结果、下一步做什么、什么时候停止
```

所以 Tool Calling 是 Agent 的基础，但不是完整 Agent。

## 5. 状态是什么

Agent 每一步都要记住一些东西，这些就是状态。

例子：

```text
goal: 我想学习 Graph RAG
plan: [检索资料, 查图谱关系, 生成任务]
evidence: [Graph RAG 文档片段]
tool_results: [RAG 搜索结果, Graph RAG 路径]
final_tasks: [先复述概念, 再跑实验, 最后写笔记]
```

没有状态管理，Agent 很容易忘记前面做过什么。

## 6. 控制流是什么

控制流决定下一步去哪。

简单控制流：

```text
计划 -> 检索 -> 评估 -> 生成 -> 结束
```

复杂控制流：

```text
计划 -> 检索 -> 评估
  如果证据够：生成答案 -> 反思 -> 结束
  如果证据不够：改写问题 -> 重新检索 -> 再评估
```

LangGraph 的核心价值就是把这种流程显式建成图。

## 7. Agent 为什么需要证据门

Agent 最大风险之一是“工具结果不够，它还硬答”。

证据门就是判断：

- 有没有证据。
- 证据是否相关。
- 证据是否足够回答问题。
- 是否应该拒答或继续检索。

证据门能减少幻觉。

## 8. Agent 为什么需要停止条件

如果没有停止条件，Agent 可能会一直调用工具。

常见停止条件：

- 达到最大步数。
- 证据已经足够。
- 用户目标已经完成。
- 多次检索没有新信息。
- 工具连续失败。

## 9. Agent 和 RAG 怎么结合

RAG 可以作为 Agent 的一个工具。

```text
Agent 目标：回答用户问题
工具 1：RAG 检索知识库
工具 2：Graph RAG 查关系路径
工具 3：Rerank 排序证据
工具 4：任务生成
```

Agent 不等于替代 RAG，而是组织 RAG。

## 10. Agent 怎么评测

不要只看最终回答。

要看：

- 工具选对了吗？
- 工具参数对了吗？
- 有没有无意义循环？
- 证据够不够？
- 最终结果是否基于证据？
- 任务是否可执行？
- 平均调用了多少步？
- 成本和延迟是否可接受？

## 11. 面试怎么讲 Agent

一个比较完整的回答：

```text
Agent 是围绕目标执行多步任务的系统。它通常包含目标、状态、工具、计划器、执行器、观察器、评估器和停止条件。工程上不能让模型无限自由发挥，要用结构化工具、状态流、最大步数、证据门和日志控制风险。RAG、Graph RAG、数据库查询都可以作为 Agent 工具。复杂 Agent 可以用 LangGraph 把流程显式建成状态图。
```

## 12. 你应该怎么练

第一轮：

1. 用默认目标运行。
2. 点击每个轨迹步骤。
3. 看输入和输出。

第二轮：

1. 关闭 Graph RAG 工具。
2. 重新运行。
3. 看工具调用轨迹少了什么。

第三轮：

1. 把证据阈值改成严格。
2. 重新运行。
3. 看 Agent 是否更容易提示证据不足。

第四轮：

1. 新增一条知识片段。
2. 让目标命中这条知识。
3. 看最终结果是否引用新知识。
