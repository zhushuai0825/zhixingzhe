项目总览

本地教学版
Agent = 大模型 + 工具 + 状态 + 控制流

单次问答只给答案，Agent 会围绕目标做计划、调用工具、观察证据、评估风险，再输出可执行结果。

这个实验室把 Agent 拆开给你看：目标怎么变成计划，计划怎么选择工具，工具输出怎么被观察，证据不足怎么拒答或改计划，最后怎么生成任务和总结。

怎么读 Agent

不要把 Agent 理解成一个神秘大脑。

状态

目标、计划、工具输出、证据、任务都属于状态。没有状态管理，Agent 很容易乱。

工具

工具是 Agent 能做事的边界。没有工具，模型只能聊天。

怎么读节点

绿色节点通常是计划或行动，蓝色节点是工具调用，橙色节点是评估。点击节点时，详情区会显示这一步吃了什么输入、吐了什么输出。

最终结果在哪里

实验台里的“最终结果”才是交付给用户看的内容；轨迹图只是解释 Agent 为什么会得到这个结果。

停止条件

Agent 必须知道什么时候结束，否则可能循环调用工具。

学习路线

Agent 建议在 RAG、Graph RAG、评测之后学习。

Agent 面试题

重点讲工程控制，不要只说“让大模型自己思考”。
