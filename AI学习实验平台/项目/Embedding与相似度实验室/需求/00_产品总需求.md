# Embedding 与相似度实验室产品总需求

## 1. 项目定位

本项目是 AI 学习实验平台中的底层能力实验室，目标是让 0 基础学习者直观看懂：

```text
为什么 RAG 要先做 Embedding，
Embedding 得到的向量是什么，
向量之间怎么计算相似度，
相似度排名怎么变成检索结果。
```

它不是完整 RAG 系统，也不是向量数据库项目，而是 RAG 学习路径中的第一块地基。

## 2. 目标用户

- 想学习 AI 应用开发的测试工程师。
- 已经知道 RAG 大概流程，但不理解 embedding 和相似度的人。
- 想从可视化界面理解“文本怎么被匹配”的学习者。

## 3. 核心学习问题

本项目必须回答：

- 文本为什么不能直接拿来做语义检索？
- token 是什么？
- embedding 向量是什么？
- 两段文本变成向量后，怎么判断谁更像谁？
- 余弦相似度、点积、欧氏距离有什么区别？
- 为什么同一个查询在不同相似度算法下排序可能不同？
- 教学语义维度 embedding 和真实 embedding 模型有什么区别？

## 4. 功能范围

### 4.1 查询输入

用户可以输入一段查询文本，例如：

```text
我想学习 RAG 里的向量数据库
```

### 4.2 候选文本编辑

系统提供默认候选文本，用户也可以改写候选文本。

候选文本用于模拟知识库片段。

### 4.3 Embedding 过程展示

系统展示：

- 查询文本的 token。
- 候选文本的 token。
- 向量维度。
- 非零维度。
- 向量条形图。

### 4.4 相似度计算

系统支持三种相似度/距离：

- 余弦相似度。
- 点积。
- 欧氏距离。

### 4.5 Top K 结果

系统按分数排序，展示最相似的候选文本。

### 4.6 匹配可视化

系统用 Canvas 展示：

```text
查询文本向量 -> 候选文本向量 -> 命中路径
```

用户能直观看到分数高的结果离查询更近或连线更强。

## 5. 非目标

当前阶段不做：

- 文档上传。
- PDF 解析。
- 文本切分。
- 向量数据库持久化。
- Rerank 模型。
- 大模型生成回答。

这些会在后续独立项目里继续做。

## 6. 成功标准

完成后，用户应该能说清楚：

```text
Embedding 是把文本变成数字向量；
相似度计算是比较两个向量；
RAG 的检索就是把问题向量和文档片段向量做相似度排序。
```
