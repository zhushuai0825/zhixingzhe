# Embedding 零基础学习手册

## 1. 先用一句话理解 Embedding

Embedding 就是：

```text
把文本、图片、音频等内容，转换成一组数字。
```

对 RAG 来说，最常见的是文本 embedding：

```text
一段文本 -> 一个向量
```

例如：

```text
“RAG 里的向量数据库是什么”
```

可能会被模型转换成：

```text
[0.12, -0.03, 0.88, 0.41, ...]
```

这组数字就是向量。

## 2. 为什么文本要变成向量

计算机不能直接理解“语义相似”。

如果只看字符串：

```text
“向量数据库”
“embedding 存储”
```

这两句话表面字符不完全一样，但语义可能很接近。

Embedding 的价值就是把语义映射到数字空间。

映射后，系统就可以用数学方法比较它们：

```text
两个向量越接近，文本语义越可能相似。
```

## 3. RAG 为什么离不开 Embedding

RAG 的检索流程是：

```text
用户问题
  -> 问题 embedding
  -> 和知识库切片 embedding 做相似度比较
  -> 找到 Top K 相关切片
  -> 把切片交给大模型回答
```

如果没有 embedding，系统只能做关键词匹配。

关键词匹配的问题是：

- 词不一样但意思一样，可能匹配不到。
- 词一样但意思不一样，可能误匹配。
- 无法很好理解上下文语义。

## 4. token 是什么

模型通常不会直接处理完整句子，而是先把文本拆成 token。

token 可以理解成模型处理文本的最小单位。

英文里 token 可能接近单词或子词：

```text
embedding database
```

中文里 token 可能是字、词，或者模型自己的子词片段。

当前教学项目为了可解释，用简化规则：

```text
英文和数字按连续片段切；
中文按单字切。
```

真实模型会使用自己的 tokenizer，例如 BPE、SentencePiece 等。

## 5. 向量维度是什么

向量维度就是数字列表的长度。

例如 4 维向量：

```text
[0.1, 0.2, 0.3, 0.4]
```

16 维向量：

```text
[0.1, 0.0, 0.2, 0.4, ... 共 16 个数字]
```

真实 embedding 模型常见维度可能是：

- 384 维。
- 768 维。
- 1024 维。
- 1536 维。
- 3072 维。

维度越高，不一定永远越好。维度高通常表达能力更强，但也会带来更多存储和计算成本。

## 6. 当前项目的教学 embedding 怎么做

当前项目用语义维度 embedding。

流程：

```text
文本
  -> tokenize
  -> 每个 token 映射到一个或多个有名字的语义维度
  -> 同一语义维度上的 token 权重累加
  -> L2 归一化
  -> 得到一个教学向量
```

它不是生产级语义模型。

但它有一个好处：

```text
你能看清楚每个 token 为什么会影响某个语义维度。
```

例如：

```text
RAG -> RAG 检索
embedding -> Embedding 向量化
向、量 -> Embedding 向量化
数、据、库 -> 向量数据库
相、似、度 -> 相似度计算
```

所以当查询文本和候选文本都命中这些维度时，它们会更相似。

## 7. 什么是 L2 归一化

向量有方向，也有长度。

L2 归一化就是把向量长度缩放到 1。

例如：

```text
[3, 4]
```

它的长度是：

```text
sqrt(3^2 + 4^2) = 5
```

归一化后：

```text
[3/5, 4/5] = [0.6, 0.8]
```

归一化的好处是：

```text
比较文本时更关注方向，而不是文本长度。
```

否则长文本可能因为 token 多，向量长度更大，在点积里占便宜。

## 8. 教学 embedding 和真实 embedding 的区别

### 8.1 教学语义维度 embedding

优点：

- 本地运行。
- 不用 API Key。
- 可解释。
- 适合学习流程。

缺点：

- 不真正理解语义。
- 维度低。
- 只能模拟 embedding 的形态，不能替代真实模型。

### 8.2 真实 embedding 模型

真实模型是通过大量数据训练出来的。

它能把语义相近的文本映射到相近位置。

例如：

```text
“怎么搭建 RAG 知识库”
“如何构建检索增强生成系统”
```

真实 embedding 模型通常能判断它们相关。

教学语义维度 embedding 未必能做到。

## 9. 真实项目里怎么使用 Embedding API

真实流程通常是：

```text
准备文本
  -> 调用 embedding API
  -> 得到向量
  -> 写入向量库
  -> 查询时再次调用 embedding API
  -> 用查询向量检索相似切片
```

重要工程点：

- 文档切片后再 embedding。
- embedding 模型要固定。
- 换模型后通常要重建索引。
- 向量维度必须和向量库字段一致。
- 大量文本要批量 embedding，不能一条条慢慢调。
- API 成本、限流、失败重试都要考虑。

## 10. 你要记住的核心

```text
Embedding 不是回答问题。
Embedding 只是把文本变成向量。

RAG 检索也不是大模型在读全文。
它是先用向量相似度找到可能相关的片段。
```
