# 页面字段与 3D 可视化说明

## 1. 查询文本查的是哪里的内容

页面里的 `查询文本` 不是去查互联网，也不是查你电脑里的全部文件。

它只查当前页面中的：

```text
候选文本集合
```

你可以把候选文本集合理解成一个迷你知识库。

真实 RAG 系统里，这个候选集合会来自：

```text
文档切分后的 chunk
  -> 做 embedding
  -> 写入向量数据库
  -> 查询时从向量数据库取出 Top K
```

当前页面为了教学，把向量库先简化成一个可编辑的文本框。

## 2. 查询文本向量是什么

查询文本向量就是：

```text
把你输入的问题变成一组数字。
```

当前项目使用教学语义维度。

例如问题：

```text
RAG 里的 embedding 和向量数据库是什么关系？
```

会命中这些维度：

- RAG 检索。
- Embedding 向量化。
- 向量数据库。
- 相似度计算。

所以它不再只是原始文字，而是一个可以计算相似度的向量。

## 3. 为什么第一条会命中

系统会把查询文本和每条候选文本都变成向量。

然后比较：

```text
查询向量 和 候选向量 的方向是否接近
```

如果某条候选文本和查询文本共同命中了更多语义维度，它的分数就会更高。

例如候选文本：

```text
向量数据库负责存储 chunk embedding，并支持 Top K 相似向量查询。
```

它和查询文本都命中了：

- 向量数据库。
- Embedding 向量化。
- RAG 检索。

所以它会排在前面。

## 4. 3D 相似度空间怎么看

3D 画布里：

- 中心点是查询向量。
- 其他点是候选文本向量。
- 越靠近中心，表示越相似。
- 橙色点是当前第一名。
- 连线越明显，表示相似度越高。

你可以：

- 拖拽画布旋转视角。
- 滚轮缩放空间。
- 鼠标放到节点上看文本和共同语义维度。

## 5. 为什么这是教学模型

真实 embedding 模型不是靠人工写语义维度。

真实模型通过大量训练数据学习语义空间。

当前项目这样设计，是为了让你看懂：

```text
文本如何变成向量
向量如何比较相似
Top K 为什么会排序
```

等你理解之后，再接真实 embedding API 或本地 embedding 模型。
