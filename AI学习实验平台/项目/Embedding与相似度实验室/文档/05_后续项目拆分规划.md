# 后续项目拆分规划

## 1. 为什么要拆项目

RAG 底层知识很多，如果全塞进一个页面，会变成一锅粥。

更适合拆成：

```text
一个核心概念 = 一个独立实验室
```

每个实验室都有：

- 前端可视化。
- 后端接口。
- 学习文档。
- 测试清单。
- 接入统一平台。

## 2. 推荐拆分

### 2.1 Embedding 与相似度实验室

当前项目。

学习：

- token。
- embedding。
- 向量。
- 归一化。
- 余弦相似度。
- 点积。
- 欧氏距离。

### 2.2 文本切分实验室

下一步建议做。

学习：

- chunk size。
- overlap。
- 按字符切分。
- 按段落切分。
- 按标题结构切分。
- 递归切分。
- 切片质量如何影响召回。

### 2.3 向量库实验室

学习：

- Chroma。
- FAISS。
- pgvector。
- collection。
- index。
- metadata。
- upsert。
- query。
- delete。

### 2.4 Rerank 实验室

学习：

- 初召回。
- 精排序。
- Cross Encoder。
- BGE reranker。
- Top K 到 Top N。
- rerank 前后排序对比。

### 2.5 RAG 检索链路整合实验室

最后整合：

```text
文本切分 -> embedding -> 向量库 -> 初召回 -> rerank -> 引用片段 -> LLM 回答
```

这个项目会把前面所有小项目串起来。
