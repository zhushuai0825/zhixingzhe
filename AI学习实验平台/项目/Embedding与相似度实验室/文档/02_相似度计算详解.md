# 相似度计算详解

## 1. 相似度解决什么问题

Embedding 把文本变成向量之后，还需要回答：

```text
哪个文档片段和用户问题最像？
```

这个问题靠相似度计算解决。

## 2. 余弦相似度

余弦相似度比较两个向量的方向。

公式：

```text
cosine(A, B) = dot(A, B) / (|A| * |B|)
```

解释：

- `dot(A, B)` 是点积。
- `|A|` 是向量 A 的长度。
- `|B|` 是向量 B 的长度。

余弦相似度范围通常是：

```text
-1 到 1
```

在很多 embedding 检索场景里，越接近 1 越相似。

## 3. 点积

点积公式：

```text
dot(A, B) = A1*B1 + A2*B2 + ... + An*Bn
```

例子：

```text
A = [0.6, 0.8]
B = [0.8, 0.6]
dot(A, B) = 0.6*0.8 + 0.8*0.6 = 0.96
```

如果向量已经归一化，点积和余弦相似度等价。

## 4. 欧氏距离

欧氏距离比较两个点之间的直线距离。

公式：

```text
distance(A, B) = sqrt((A1-B1)^2 + (A2-B2)^2 + ... + (An-Bn)^2)
```

距离越小越相似。

这点和余弦、点积相反：

```text
余弦/点积：越大越相似。
欧氏距离：越小越相似。
```

## 5. 什么时候用哪种

常见选择：

- 文本 embedding 检索：常用余弦相似度。
- 向量已归一化：点积也常用，因为计算快。
- 图像、聚类、传统向量空间：有时会用欧氏距离。

实际要看模型和向量库推荐的指标。

例如某些 embedding 模型明确建议使用 cosine，就优先使用 cosine。

## 6. Top K 是什么

Top K 表示取前 K 个最相似结果。

例如：

```text
Top K = 5
```

意思是：

```text
从所有候选片段里找出最相关的 5 个。
```

RAG 常见流程就是：

```text
问题向量
  -> 和所有切片向量比较
  -> 取 Top K
  -> 拼进 prompt
```

## 7. 为什么 Top K 不是越大越好

Top K 太小：

- 可能漏掉关键证据。

Top K 太大：

- prompt 变长。
- 噪声变多。
- 大模型更容易被不相关内容干扰。
- 成本更高。

所以 Top K 是一个需要调参的工程参数。

## 8. 相似度高是否一定正确

不一定。

原因：

- embedding 模型可能理解错。
- 文档切片可能切得不好。
- 查询问题可能太模糊。
- Top K 召回可能混入噪声。
- 相似不等于能回答。

这就是为什么后面还要学：

```text
文本切分
向量数据库
Hybrid 检索
Rerank
答案引用
```

## 9. Rerank 和相似度有什么关系

初步向量检索通常速度快，但排序不一定最精细。

Rerank 是第二阶段：

```text
先用向量相似度召回 Top 20
再用 rerank 模型精排成 Top 5
```

向量检索负责“找一批可能相关的”。

Rerank 负责“从这批里重新判断谁最相关”。

## 10. 当前项目你应该怎么练

1. 修改查询文本。
2. 修改候选文本。
3. 切换余弦、点积、欧氏距离。
4. 观察 Top K 排序是否变化。
5. 看共享 token 和画布连线。
6. 思考为什么某个结果排第一。

能做到这一步，说明你已经不是只会“调用 RAG”，而是在理解检索底层。
