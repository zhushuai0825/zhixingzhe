# 文本切分实验室产品总需求

## 1. 项目定位

文本切分实验室是 RAG 底层能力学习项目，目标是让学习者看懂：

```text
长文档为什么不能直接 embedding，
为什么要切成 chunk，
chunk size 和 overlap 如何影响检索效果。
```

## 2. 目标用户

- 正在学习 RAG 的测试工程师。
- 已经知道 embedding，但不理解文档切分的人。
- 想看懂 RAG 检索为什么有时召回不到正确答案的人。

## 3. 必须回答的问题

- 什么是 chunk？
- chunk size 是什么？
- overlap 是什么？
- 为什么 overlap 可以保护边界信息？
- 为什么 overlap 太大会浪费？
- 按字符切分、按段落切分、递归切分有什么区别？
- 怎么判断切片质量好不好？

## 4. 功能范围

### 4.1 文本输入

用户可以输入一段长文本，模拟 PDF、Word、网页解析后的内容。

### 4.2 参数调整

用户可以调整：

- 切分策略。
- chunk size。
- overlap。

### 4.3 切片展示

系统展示：

- chunk 数量。
- 每个 chunk 的内容。
- 每个 chunk 的长度。
- 每个 chunk 的原文位置。
- 每个 chunk 的 overlap 前缀长度。

### 4.4 质量提示

系统展示：

- 平均长度。
- 太短切片数量。
- 太长切片数量。
- 边界风险数量。

### 4.5 可视化

系统用 Canvas 展示原文轴线和 chunk 色块，让用户直观看到：

- chunk 覆盖了原文哪里。
- 相邻 chunk 是否重叠。
- 哪些 chunk 可能切得不好。

## 5. 非目标

当前不做：

- PDF 上传。
- Word 解析。
- OCR。
- embedding。
- 向量数据库。
- Rerank。
- 大模型回答。

这些会在后续项目里单独学习。
