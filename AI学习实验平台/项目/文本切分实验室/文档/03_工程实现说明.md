# 工程实现说明

## 1. 项目结构

```text
文本切分实验室/
  README.md
  前端原型/
  后端/
  文档/
  需求/
  规划/
  学习资料/
```

## 2. 后端接口

端口：

```text
8040
```

接口：

| 接口 | 作用 |
| --- | --- |
| `GET /health` | 检查服务是否启动。 |
| `GET /api/example` | 返回默认示例文本。 |
| `POST /api/split` | 根据参数执行切分。 |

## 3. POST /api/split 输入

```json
{
  "text": "长文档文本",
  "strategy": "recursive",
  "chunkSize": 260,
  "overlap": 60
}
```

## 4. 输出

返回：

- strategy。
- chunkSize。
- overlap。
- sourceLength。
- chunkCount。
- chunks。
- quality。

每个 chunk 包含：

- id。
- text。
- start。
- end。
- length。
- overlapPrefix。

`length` 表示当前 chunk 的总长度，包含 overlap。修复后的实现会保证每个 chunk 总长度不超过 `chunkSize`。例如 `chunkSize=260`、`overlap=60` 时，后续 chunk 最多是 60 字重叠内容加 200 字新内容。

## 5. 前端画布含义

画布横轴表示原文位置。

每个色块表示一个 chunk。

深色区域表示 overlap。

如果色块颜色偏橙，说明可能有边界风险或过短风险。

## 6. 当前局限

- 当前按字符长度计算，不是 token 长度。
- 当前只处理纯文本，不处理 PDF/Word。
- 当前不保存切片数据。
- 当前不做 embedding。

这些都会在后续项目里继续扩展。
