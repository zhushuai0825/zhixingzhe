# 测试验收清单

## 1. 语法检查

```bash
node --check 前端原型/app.js
python3 -m py_compile 后端/app/main.py
```

## 2. 后端检查

```bash
curl http://127.0.0.1:8040/health
```

期望：

```json
{
  "status": "ok"
}
```

## 3. 切分接口检查

```bash
curl -X POST http://127.0.0.1:8040/api/split \
  -H 'Content-Type: application/json' \
  -d '{"text":"第一段内容。第二段内容。第三段内容。","strategy":"character","chunkSize":10,"overlap":2}'
```

期望：

- 返回 chunkCount。
- chunks 不为空。
- 每个 chunk 有 text、start、end、length。

## 4. 前端检查

打开：

```text
http://127.0.0.1:8099/平台前端/
```

切换到：

```text
文本切分实验室
```

检查：

- 页面能打开。
- 可以修改 chunk size。
- 可以修改 overlap。
- 可以切换三种策略。
- 修改参数后会自动重新切分，也可以点击 `运行切分` 手动触发。
- 每个 chunk 的总长度不应超过当前 chunk size。
- 如果递归切分数量暂时不变，需要查看页面里的“为什么数量可能没变”：递归切分会优先保留自然边界，只有参数跨过边界阈值时数量才明显变化。
- 切片列表更新。
- Canvas 边界图更新。
- 鼠标悬停 chunk 能看到内容。
