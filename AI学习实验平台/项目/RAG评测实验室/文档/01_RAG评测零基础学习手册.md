# RAG 评测零基础学习手册

## 1. 为什么要评测

RAG 系统不是“回答看起来像样”就算成功。

一个回答可能：

- 看起来很流畅，但证据没命中。
- 引用了文档，但引用不能支撑答案。
- 检索到了正确证据，但大模型回答时编了额外内容。
- 知识库没有答案，却硬编一个答案。

所以要评测。

## 2. 评测集是什么

评测集是一组标准题目：

```text
问题
标准证据
标准答案
是否应该拒答
```

例子：

```text
问题：Rerank 为什么能提升 RAG 准确率？
标准证据：d4
标准答案：Rerank 会重新排序候选证据，把更能回答问题的 chunk 放前面。
是否应该拒答：否
```

不可回答问题也要放进评测集：

```text
问题：当前知识库里有没有公司 2026 年薪酬制度？
标准证据：无
标准答案：知识库没有足够依据，应该拒答。
是否应该拒答：是
```

## 3. Recall@K

Recall@K 看的是：标准证据有没有出现在 Top K 检索结果里。

如果标准证据是 `d4`：

```text
Top 3 = d2, d4, d1
```

那么 Recall@3 命中。

如果：

```text
Top 3 = d2, d1, d5
```

就没命中。

口语理解：系统有没有把该找的资料找回来。

## 4. MRR

MRR 看的是：第一个正确证据排在第几名。

```text
正确证据排第 1：得 1
正确证据排第 2：得 1/2
正确证据排第 3：得 1/3
没找到：得 0
```

口语理解：不只是找回来，还要排得靠前。

## 5. 引用准确率

引用准确率看的是：回答引用的文档是不是真的支撑答案。

如果系统引用了 `d4`，而 `d4` 是标准证据，引用准确。

如果系统引用了 `d6 Agent 工具调用` 来回答 Rerank，引用就不准。

## 6. 忠实度

忠实度看的是：回答有没有严格基于证据。

如果证据只说：

```text
Rerank 会重新排序候选证据。
```

但模型回答：

```text
Rerank 会自动训练 embedding 模型，并保证回答 100% 正确。
```

这就是不忠实，因为证据没有支持这些内容。

## 7. 拒答正确性

企业 RAG 必须会拒答。

如果知识库没有答案，正确行为是：

```text
当前知识库没有足够依据回答这个问题。
```

错误行为是：

```text
公司 2026 年薪酬制度是……
```

拒答评测可以发现系统是否喜欢瞎编。

## 8. 指标差了怎么优化

如果 Recall@K 低：

- 调整 chunk size。
- 调整 overlap。
- 换 embedding 模型。
- 加 Hybrid 检索。
- 提高 Top K。

如果 MRR 低：

- 加 Rerank。
- 优化 query 改写。
- 优化 metadata 过滤。

如果引用准确率低：

- 检查 source、page、chunk_id。
- prompt 要求只引用命中证据。
- 前端展示引用详情。

如果忠实度低：

- 加证据门。
- 不足时拒答。
- prompt 明确“只能基于引用回答”。

## 9. 面试回答模板

如果面试官问“RAG 怎么评测”，可以这样答：

```text
我会分层评测。
检索层看 Recall@K、MRR、NDCG，判断标准证据有没有被找回并排到前面。
回答层看引用准确率、忠实度、答案覆盖率，判断回答是否真的被证据支撑。
安全层看拒答正确率，专门测试知识库没有答案时系统是否会编造。
发现问题后按层优化：检索差调切分、embedding、Hybrid、Top K；排序差加 rerank；回答不忠实则调 prompt、证据门和拒答策略。
```
