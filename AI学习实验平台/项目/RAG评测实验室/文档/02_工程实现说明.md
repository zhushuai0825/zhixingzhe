# 工程实现说明

## 1. 当前版本

当前版本是纯前端教学版，不调用真实大模型。

它用本地数据模拟：

- 文档库。
- 评测问题。
- 检索结果。
- 系统回答。
- 指标计算。

目标是让你先看懂评测逻辑。

## 2. 数据结构

文档：

```js
{
  id: "d4",
  title: "Rerank 提准",
  source: "rerank.md",
  text: "Rerank 位于向量召回之后..."
}
```

评测问题：

```js
{
  id: "q3",
  question: "Rerank 为什么能提升 RAG 准确率？",
  goldDocs: ["d4"],
  goldAnswer: "Rerank 会重新排序向量召回的候选证据...",
  shouldAnswer: true
}
```

## 3. 指标怎么计算

### Recall@K

```text
标准证据是否出现在 Top K 里
```

### MRR

```text
第一个标准证据排第几名
```

### 引用准确率

```text
引用中的标准证据数量 / 引用总数
```

### 忠实度

教学版用规则模拟：如果回答引用了标准证据，就认为忠实度较高；如果没有标准证据还强答，就认为忠实度低。

### 拒答正确性

```text
该拒答时拒答 = 1
不该拒答却拒答 = 0
```

## 4. 后续真实版本

后续可以接入真实 RAG 日志：

```text
query
retrieved_chunks
reranked_chunks
answer
citations
latency
cost
feedback
```

然后计算：

- Recall@K。
- MRR。
- NDCG。
- Faithfulness。
- Answer Relevance。
- Citation Precision。
- Refusal Accuracy。
- Latency。
- Cost。

## 5. 为什么要做成实验室

因为 RAG 优化不能靠感觉。

你改了 chunk size、embedding、Top K、Rerank、prompt 后，必须能回答：

```text
指标有没有变好？
哪个指标变好了？
哪个指标变坏了？
为什么？
```
