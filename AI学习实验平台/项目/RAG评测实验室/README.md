# RAG 评测实验室

这个实验室用于学习如何判断一个 RAG 系统到底准不准。

很多初学者会以为“模型能回答”就代表系统做好了，但企业 RAG 更看重：

- 检索有没有找回标准证据。
- 标准证据排得靠不靠前。
- 回答有没有引用正确来源。
- 回答是否严格基于证据。
- 证据不足时是否能拒答。

## 核心流程

```text
准备评测集
  -> 运行 RAG 检索和回答
  -> 计算检索指标
  -> 计算回答指标
  -> 诊断失败原因
  -> 优化切分、Embedding、向量库、Rerank、Prompt
```

## 当前能力

- 选择内置评测问题。
- 新增自定义评测问题。
- 设置标准证据 ID。
- 切换 RAG 系统版本。
- 调整检索 Top K。
- 查看检索命中结果。
- 查看系统回答和标准答案。
- 计算 Recall@K、MRR、引用准确率、忠实度、答案覆盖、拒答正确性。
- 根据短板输出优化建议。
- 用 Canvas 展示指标短板。
