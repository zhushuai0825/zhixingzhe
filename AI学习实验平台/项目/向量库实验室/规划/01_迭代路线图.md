# 迭代路线图

## V1：教学内存向量库

已完成能力：

- 示例 collection。
- Upsert。
- Query Top K。
- Delete。
- Reset。
- Metadata source 过滤。
- 向量维度预览。
- 3D 命中空间。

目标：先把向量库核心行为看懂。

## V2：接入真实 Embedding

计划能力：

- 支持本地 sentence-transformers。
- 支持通义千问 / DeepSeek / OpenAI 兼容 Embedding API。
- 展示真实向量维度，但默认只显示摘要和前 N 个维度，避免页面太乱。

目标：从教学向量升级到真实语义向量。

## V3：接入 Chroma 或 pgvector

计划能力：

- Chroma：最适合入门。
- PostgreSQL + pgvector：适合和业务数据一起存。
- 保留同样的前端体验，让你看到“接口不变，底层可替换”。

目标：理解向量数据库的真实存储和索引。

## V4：接入 Rerank

计划能力：

- 向量库先召回 Top K。
- Rerank 再按更精细语义重新排序。
- 对比“只向量检索”和“向量检索 + Rerank”的差异。

目标：进入企业级 RAG 的检索后优化。

## V5：和知行者 AI 实验室合流

计划能力：

- 上传文档。
- 切分 chunk。
- 生成 embedding。
- 写入向量库。
- 检索引用。
- 大模型回答。
- Agent 调用检索工具。

目标：形成完整 RAG + Agent 学习系统。
