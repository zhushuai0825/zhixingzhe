# 后续迭代计划

## 1. 接入真实 Embedding

当前教学 embedding 适合看懂原理，但不适合真实业务。

下一步可以接：

- 本地 sentence-transformers。
- 通义千问 embedding API。
- OpenAI 兼容 embedding API。
- BGE / M3E 等中文向量模型。

## 2. 接入 PostgreSQL + pgvector

你前面已经希望主数据库用 PostgreSQL。对于向量库，也可以用 pgvector。

优点：

- 业务数据和向量数据可以在同一个数据库里管理。
- SQL 容易理解。
- 本地 Docker 部署方便。

适合学习：

- 表结构设计。
- 向量字段。
- metadata JSONB。
- 向量索引。
- SQL 相似度查询。

## 3. 接入 Chroma

Chroma 更适合 AI 应用入门。

适合学习：

- collection API。
- add / upsert / query / delete。
- metadata filter。
- 本地持久化。

## 4. 接入 Milvus

Milvus 更偏企业级。

适合后期学习：

- 大规模向量索引。
- 分区。
- 高性能检索。
- 集群部署。

## 5. 新增 Rerank 实验室

向量库返回 Top K 后，还会有不够准的问题。

Rerank 的作用是：

```text
向量库粗召回
  ↓
Rerank 精排序
  ↓
选出最适合喂给大模型的证据
```

这是下一阶段最值得做的模块。
