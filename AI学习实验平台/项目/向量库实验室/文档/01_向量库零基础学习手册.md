# 向量库零基础学习手册

## 1. 向量库到底是什么

向量库是专门用来存储和检索向量的数据库。

普通数据库擅长回答：

- id 等于多少。
- 日期在哪个范围。
- 状态是不是成功。
- 名字是不是某个值。

向量库擅长回答：

- 哪些文本和这个问题语义最像。
- 哪些图片和这张图片最像。
- 哪些商品和当前商品最像。
- 哪些代码片段和当前需求最相关。

RAG 里最常见的是文本向量库：每个 chunk 被 embedding 成向量后写入向量库，用户提问时问题也被 embedding 成向量，再找最相似的 chunk。

## 2. 向量库不是大模型

向量库不会自己生成答案。

它做的是：

```text
输入：query vector
处理：和 collection 里的 chunk vector 比相似度
输出：Top K chunk
```

大模型做的是：

```text
输入：用户问题 + Top K chunk
处理：理解上下文并生成语言
输出：最终回答
```

所以 RAG 的关键分工是：

- 向量库找证据。
- 大模型基于证据回答。

## 3. 为什么 RAG 必须有向量库

如果没有向量库，系统每次回答都要扫描全部文档。文档一多就会非常慢，而且很难找到语义相关的内容。

向量库的价值是：

- 可以把文档提前处理好。
- 查询时只找最相关的 Top K。
- 可以按 metadata 过滤范围。
- 可以保存引用来源。
- 可以支撑企业级知识库问答。

## 4. 一条向量记录存什么

一条记录通常包含：

```text
id
text
embedding
metadata
```

解释：

- `id`：唯一编号，方便覆盖和删除。
- `text`：chunk 原文，大模型回答时要引用它。
- `embedding`：文本向量，用来计算相似度。
- `metadata`：来源信息，比如文件名、页码、章节、创建人、权限。

## 5. Collection 是什么

Collection 可以理解为一组同类向量记录。

例如：

- `product_manual_chunks`：产品手册切片。
- `test_case_chunks`：测试用例切片。
- `company_policy_chunks`：公司制度切片。
- `rag_learning_chunks`：当前实验室示例切片。

一个 RAG 系统可以有一个 collection，也可以有多个 collection。

## 6. Upsert 是什么

Upsert = Update + Insert。

意思是：

- 如果 id 不存在，就新增。
- 如果 id 已存在，就覆盖。

为什么向量库常用 upsert？

因为文档可能会更新。比如 `chunk-1` 的原文变了，它的 embedding 也应该重新计算并覆盖旧记录。

## 7. Query 是什么

Query 的流程：

```text
用户输入问题
  ↓
问题变成 query embedding
  ↓
query embedding 和 collection 里每个 chunk embedding 比较
  ↓
按相似度排序
  ↓
返回 Top K
```

前端页面里“查询文本”查询的是当前 collection，不是搜索互联网，也不是扫描文件夹。

## 8. Top K 是什么

Top K 表示返回前 K 条最相似的记录。

例如 Top K = 3：

```text
第 1 条：score 0.82
第 2 条：score 0.71
第 3 条：score 0.66
```

K 不是越大越好：

- 太小：可能漏掉答案。
- 太大：会把噪声带给大模型。

企业 RAG 常见做法是先召回较多，例如 Top 20，再用 Rerank 选出最有用的 3 到 8 条。

## 9. Metadata 为什么重要

Metadata 不是用来算语义的，它是用来管理和过滤的。

常见 metadata：

- `source`：文件名。
- `page`：页码。
- `section`：章节。
- `created_at`：入库时间。
- `owner`：所属人。
- `permission`：权限范围。

例子：

用户问“接口超时怎么处理”，如果不加过滤，系统可能从所有文档里找。如果加上 `source=test_manual.md`，就只在测试手册里找。

## 10. Index 是什么

真实向量库不会每次都暴力比较全部向量。数据量大时会建立索引，加速近似最近邻搜索。

你现在先不用急着学复杂索引，只要知道：

- 小数据可以直接算相似度。
- 大数据需要索引。
- 索引会在速度、内存、准确率之间做取舍。

## 11. 当前实验室怎么学习

建议你按这个顺序操作：

1. 打开“向量库实验台”。
2. 看默认 collection 中有哪些记录。
3. 用默认问题查询一次。
4. 看 Top K 命中结果。
5. 点中某条记录，看它的向量维度。
6. 自己新增一条记录。
7. 再查询，观察新记录是否被命中。
8. 删除一条记录，再查询，观察它是否消失。
9. 切到“命中可视化”，拖拽、缩放，看 query 和 chunk 的距离。

## 12. 和其他项目的关系

在学习平台里，它的位置是：

```text
文本切分实验室
  ↓
Embedding 与相似度实验室
  ↓
向量库实验室
  ↓
Rerank 实验室
  ↓
知行者 AI 实验室
```

你现在学的是 RAG 最核心的中间层：把 chunk embedding 存起来，并能在提问时找回来。
