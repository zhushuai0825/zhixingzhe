# Rerank 重排序实验室产品总需求

## 1. 项目目标

让学习者理解企业 RAG 为什么需要 Rerank，并能通过可交互实验看到“向量召回顺序”和“最终证据顺序”的差异。

## 2. 目标用户

- 正在学习 RAG 的测试工程师。
- AI 应用开发初学者。
- 想准备 RAG 面试的人。

## 3. 核心功能

- 输入用户问题。
- 调整向量召回 Top K。
- 调整最终保留 Top N。
- 切换 Rerank 策略。
- 展示向量召回列表。
- 展示 Rerank 后证据列表。
- 点击候选查看分数解释。
- 用 Canvas 展示排序前后变化。
- 提供学习路线和面试题。

## 4. 成功标准

- 用户能说清 Rerank 在 RAG 流程中的位置。
- 用户能解释为什么向量库命中不等于最终答案正确。
- 用户能解释 Top K、Top N 和成本之间的关系。
- 用户能理解证据不足时不能硬生成。
