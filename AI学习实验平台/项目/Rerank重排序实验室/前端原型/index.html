本地教学版
为什么召回后还要重新排序

向量库负责找一批可能相关的证据，Rerank 负责把最能回答问题的证据排到前面。

向量相似度擅长大范围召回，但它可能把“主题相似但没回答问题”的片段排前面。 Rerank 会重新阅读 query 和 chunk 的匹配关系，综合问题词、答案词、来源质量和噪声惩罚，决定哪些证据真正值得进入 prompt。

01Vector Recall

向量库先召回 Top K 候选证据。

02Rerank

逐条判断 query 和 chunk 是否真的匹配。

03Evidence Gate

低分证据不进入 prompt，证据不足时拒答。

04Generate

大模型只基于最终证据回答。

排序前后可视化

这是动态排序流向图，不是 3D 向量空间。左边是向量召回顺序，右边是 Rerank 后顺序，移动点展示每个候选怎么换位。

怎么看这张图

它展示排序变化，不展示 embedding 坐标。

移动点

小圆点会从左侧初筛名次移动到右侧重排名次，表示同一个 chunk 的名次变化。

为什么不是 3D

Rerank 重点不是看向量空间位置，而是看同一批候选在“召回分”和“能否回答问题”两种标准下如何换位，所以用排序流向图更直观。

上升

如果 chunk 在右侧位置更靠上，说明 reranker 认为它更能回答问题。

下降

如果 chunk 主题相似但没说到答案，它可能被降权。

被过滤

最终 Top N 之外的候选通常不进入 prompt，避免上下文噪声。
