# 迭代路线图

## V0：文档和静态前端

目标：

- 讲清 ItemCF。
- 做出可交互演示。

范围：

- 学习手册。
- 手算案例。
- 用户-物品矩阵。
- 物品相似度矩阵。
- 推荐分解释。

验收：

- 不接后端也能学习完整算法流程。
- 用户可以自己复算一个推荐结果。

## V1：算法函数和测试

目标：

- 把前端里的核心算法抽离成可测试函数。

范围：

- 独立 `itemcf.js`。
- 增加若干输入输出样例。
- 用简单脚本验证相似度和推荐分。

验收：

- 修改数据后能快速确认计算没有错。
- 文档中的手算结果和程序结果一致。

## V2：后端 API

目标：

- 用后端提供推荐接口。

技术：

- Python。
- FastAPI。
- SQLite。

功能：

- 用户列表。
- 物品列表。
- 行为列表。
- 推荐接口。
- 推荐解释接口。

验收：

- 前端可以切换本地计算和后端计算。
- 后端返回推荐结果和贡献明细。

## V3：真实数据和评估

目标：

- 从学习 Demo 走向可评估推荐系统。

范围：

- 导入 MovieLens 小数据集。
- 划分训练集和测试集。
- 计算 Precision@K、Recall@K、Coverage。
- 对比热门推荐。

验收：

- 能说明 ItemCF 是否比热门推荐更好。
- 能解释指标变化原因。

## V4：更多算法对比

目标：

- 建立推荐算法学习地图。

范围：

- UserCF。
- 热门推荐。
- 基于内容推荐。
- 矩阵分解入门。

验收：

- 同一批数据可以切换算法观察差异。
