# 产品总需求：ItemCF 学习实验室

## 1. 产品定位

ItemCF 学习实验室是一个帮助学习者理解“基于物品的协同过滤”的可视化项目。

它不是完整电商推荐系统，而是一个把推荐算法拆开、运行、观察和复盘的学习工作台。

一句话定位：

> 通过小数据集、矩阵可视化和推荐分拆解，把 ItemCF 从抽象公式变成可观察的计算过程。

## 2. 目标用户

第一目标用户：

- 想学习推荐算法基础。
- 已经了解一点 Python 或 JavaScript。
- 害怕一上来就被矩阵、公式和工程架构淹没。
- 希望看到“一个推荐结果到底怎么算出来”。

## 3. 学习目标

帮助用户掌握：

- 协同过滤的基本思想。
- UserCF 和 ItemCF 的区别。
- 用户-物品行为矩阵。
- 物品共现矩阵。
- 物品相似度矩阵。
- Top N 推荐分计算。
- 过滤已交互物品。
- 冷启动、热门偏置和稀疏数据问题。

## 4. 核心模块

第一版只保留必要模块：

1. 项目总览。
2. 算法实验台。
3. 矩阵可视化。
4. 推荐解释。
5. 学习路线。

暂不做：

- 登录和权限。
- 真实数据库。
- 多算法混合推荐。
- 模型训练平台。
- 复杂运营后台。

## 5. 核心原则

### 5.1 学习优先

每个功能都要回答三个问题：

- 这个概念是什么？
- 它在 ItemCF 里怎么计算？
- 我怎么知道计算结果对不对？

### 5.2 可观察

推荐结果不能只给商品列表。

必须展示：

- 用户喜欢过哪些物品。
- 候选物品来自哪些相似物品。
- 每个候选物品的推荐分怎么累加。
- 哪些物品因为用户已经喜欢过而被过滤。

### 5.3 先简单后复杂

学习顺序：

```text
显式小数据集
  -> 共现矩阵
  -> 相似度矩阵
  -> Top N 推荐
  -> 评估指标
  -> 后端和真实数据
```

开发顺序：

```text
文档
  -> 静态前端原型
  -> 算法函数抽离
  -> 后端 API
  -> 数据库存储
  -> 更多推荐算法对比
```

## 6. 第一版 MVP

第一版 MVP 必须完成：

- 清晰的首页和学习路线。
- ItemCF 核心概念说明。
- 内置学习数据集。
- 用户-物品行为矩阵。
- 物品相似度矩阵。
- 推荐分计算拆解。
- 切换目标用户后结果实时更新。

验收：

- 用户能说清 ItemCF 推荐来自“相似物品”。
- 用户能看懂一个候选物品的分数累加过程。
- 用户能区分“相似度”和“推荐分”。
- 不接后端也能完成算法演示。
