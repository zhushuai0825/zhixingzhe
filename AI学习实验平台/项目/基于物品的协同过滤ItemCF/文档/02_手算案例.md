# ItemCF 手算案例

这份文档用一个小数据集手算一轮推荐。

## 1. 数据集

物品：

| ID | 名称 |
| --- | --- |
| I1 | 科幻电影 |
| I2 | 太空纪录片 |
| I3 | 机器学习入门 |
| I4 | Python 实战 |
| I5 | 产品经理方法 |

用户喜欢行为：

| 用户 | 喜欢物品 |
| --- | --- |
| U1 | I1, I2, I3 |
| U2 | I1, I2 |
| U3 | I2, I3, I4 |
| U4 | I3, I4 |
| U5 | I4, I5 |

现在要给 U2 推荐。

U2 已经喜欢：

```text
I1 科幻电影
I2 太空纪录片
```

所以候选物品是：

```text
I3 机器学习入门
I4 Python 实战
I5 产品经理方法
```

## 2. 统计每个物品被多少用户喜欢

| 物品 | 喜欢用户数 |
| --- | ---: |
| I1 | 2 |
| I2 | 3 |
| I3 | 3 |
| I4 | 3 |
| I5 | 1 |

## 3. 统计关键共现

只需要算候选物品和 U2 已喜欢物品的关系。

### I3 和 I1

同时喜欢 I3、I1 的用户：

```text
U1
```

所以：

```text
co_count(I3, I1) = 1
```

相似度：

```text
sim(I3, I1) = 1 / sqrt(3 * 2) = 0.408
```

### I3 和 I2

同时喜欢 I3、I2 的用户：

```text
U1, U3
```

相似度：

```text
sim(I3, I2) = 2 / sqrt(3 * 3) = 0.667
```

### I4 和 I1

没有用户同时喜欢 I4、I1：

```text
sim(I4, I1) = 0
```

### I4 和 I2

同时喜欢 I4、I2 的用户：

```text
U3
```

相似度：

```text
sim(I4, I2) = 1 / sqrt(3 * 3) = 0.333
```

### I5 和 I1 / I2

I5 只被 U5 喜欢，而 U5 没有喜欢 I1、I2：

```text
sim(I5, I1) = 0
sim(I5, I2) = 0
```

## 4. 计算 U2 的候选推荐分

U2 喜欢 I1、I2，所以：

```text
score(U2, I3) = sim(I3, I1) + sim(I3, I2)
              = 0.408 + 0.667
              = 1.075
```

```text
score(U2, I4) = sim(I4, I1) + sim(I4, I2)
              = 0 + 0.333
              = 0.333
```

```text
score(U2, I5) = sim(I5, I1) + sim(I5, I2)
              = 0 + 0
              = 0
```

## 5. 推荐结果

排序后：

| 排名 | 推荐物品 | 分数 | 主要原因 |
| --- | --- | ---: | --- |
| 1 | I3 机器学习入门 | 1.075 | 同时接近 I1 和 I2，尤其和 I2 相似 |
| 2 | I4 Python 实战 | 0.333 | 和 I2 有弱相似 |
| 3 | I5 产品经理方法 | 0.000 | 没有相似来源 |

实际推荐时通常不会展示 0 分物品。
