# 基于物品的协同过滤 ItemCF 学习实验室

这是一个面向推荐算法入门的学习型项目。当前学习重点是 ItemCF 本身：用户行为怎么形成物品共现、相似度怎么算、推荐分怎么累加、结果为什么能解释。项目里保留过 PostgreSQL 和 FastAPI 历史能力，但现在不把后端和数据库作为必学内容。

核心流程：

```text
用户/物品/行为数据
  -> 用户-物品行为矩阵
  -> 物品共现统计
  -> 物品相似度矩阵
  -> 根据用户已喜欢物品加权求和
  -> 过滤已喜欢物品
  -> 输出 Top N 推荐
  -> 推荐匹配过程可视化
  -> 3D 物品相似度空间可视化
```

## 当前阶段

当前已经完成可编辑数据学习闭环：

- 页面可以保存用户、物品、喜欢行为。
- 默认种子数据包含 6 个用户、31 个物品、49 条喜欢行为。
- 浏览器本地算法可以直接计算推荐，适合先学习算法。
- PostgreSQL 和 FastAPI 是可选历史扩展，不影响当前学习。
- 展示用户-物品矩阵、共现矩阵、相似度矩阵。
- 展示 Top N 推荐、候选得分、贡献明细。
- 展示推荐可视化台，用连线方式观察用户历史如何命中候选物品。
- 展示 3D 物品相似度空间。

## 数据说明

当前数据不是向量数据库数据。

ItemCF 的核心数据是：

- 用户。
- 物品。
- 用户对物品的行为。

这些更适合存在 PostgreSQL。

向量数据库适合 RAG 那种“文本语义相似度检索”。ItemCF 这里的相似度来自用户行为共现，不需要 Chroma。

3D 可视化展示的是“物品相似度空间”，不是 Chroma 向量空间。

推荐可视化台展示的是“推荐匹配路径”：

- 左侧：目标用户和已喜欢物品。
- 中间：用户历史到候选物品的相似度连线。
- 右侧：Top N 推荐结果。
- 下方：数据从行为读取、相似度计算、得分累加到过滤排序的过程。

这个页面可以直接新增或删除当前用户的喜欢行为，方便你观察推荐结果如何随行为变化。鼠标悬停画布节点时，可以看到完整物品名和说明。

也可以创建自定义物品并让当前用户喜欢它。算法可以支持自定义物品，但新物品刚创建时属于冷启动：如果只有一个用户喜欢它，它还没有和其他物品形成足够共现关系，推荐分通常会很弱。等多个用户都喜欢过它以后，它才更容易进入推荐链路。

## 当前边界

- 数据量仍是学习级小数据，默认演示数据已扩展到 31 个学习物品。
- 行为只有“喜欢/未喜欢”，暂不处理评分、时序和负反馈。
- 相似度使用 ItemCF 常见的余弦归一化共现公式。
- 3D 坐标是根据相似度矩阵生成的教学可视化，不是机器学习 embedding。
