# 推荐系统核心概念

## 1. 推荐系统在做什么

推荐系统的目标是根据已有信息，猜用户接下来可能喜欢什么。

已有信息通常包括：

- 用户信息：用户是谁、角色、画像。
- 物品信息：标题、标签、分类、内容。
- 行为信息：点击、收藏、购买、学习、评分。

输出通常是：

```text
Top N 推荐列表
```

## 2. 标签/内容推荐

标签推荐看的是物品内容。

例子：

```text
用户喜欢：RAG 入门、Embedding 与相似度
系统发现：用户偏好 AI、向量、检索
推荐：向量数据库、Graph RAG
```

优点：

- 新物品没有行为也能推荐。
- 推荐理由清晰。
- 适合冷启动。

缺点：

- 容易推荐得太窄。
- 依赖标签质量。

### 2.1 一步一步怎么算

1. 给每个物品打标签。
2. 找到用户喜欢过的物品。
3. 把这些物品的标签累加成用户画像。
4. 遍历用户没看过的候选物品。
5. 候选物品标签和用户画像越重合，推荐分越高。

示例：

```text
用户喜欢：
- RAG 入门：AI、RAG、检索
- 向量数据库：AI、向量、数据库

用户画像：
AI = 2
RAG = 1
检索 = 1
向量 = 1
数据库 = 1

候选 Graph RAG：
AI、RAG、图谱

它命中 AI 和 RAG，所以可以推荐。
```

### 2.2 什么时候优先用标签推荐

- 新物品刚上线，没有用户行为。
- 业务需要强解释，比如“因为你喜欢 AI 和 RAG”。
- 用户行为很少，但物品内容很清楚。
- 需要和人工运营策略结合。

## 3. ItemCF

ItemCF 看的是物品共现。

如果很多用户同时喜欢物品 A 和物品 B，那么 A 和 B 相似。

推荐逻辑：

```text
当前用户喜欢 A
A 和 B 相似
用户还没喜欢 B
推荐 B
```

它不需要理解物品标题，只看行为。

### 3.1 一步一步怎么算

1. 准备用户行为表。
2. 统计每个物品被多少用户喜欢。
3. 统计两个物品被同一个用户同时喜欢的次数。
4. 用归一化公式计算物品相似度。
5. 对目标用户，把他喜欢过的物品作为来源。
6. 找来源物品的相似物品，累加推荐分。
7. 过滤用户已经喜欢过的物品。
8. 按推荐分排序输出 Top N。

公式：

```text
sim(i, j) = co_count(i, j) / sqrt(count(i) * count(j))
```

### 3.2 为什么要归一化

热门物品容易和所有东西共现。

如果只看共现次数，热门物品会霸榜。

归一化会除以两个物品各自的热度，让“共同喜欢比例”更公平。

### 3.3 ItemCF 的典型坑

- 新物品没人喜欢过，算不出相似度。
- 新用户没有行为，无法推荐。
- 热门物品容易过度推荐。
- 行为类型要区分权重，点击、收藏、购买不能完全等价。

## 4. UserCF

UserCF 看的是用户相似。

如果用户 A 和用户 B 喜欢过很多相同物品，那么他们相似。

推荐逻辑：

```text
用户 A 和用户 B 兴趣相似
用户 B 喜欢 C
用户 A 还没喜欢 C
给用户 A 推荐 C
```

### 4.1 一步一步怎么算

1. 准备用户行为表。
2. 把每个用户喜欢过的物品变成集合。
3. 计算目标用户和其他用户的相似度。
4. 找最相似的一批用户。
5. 收集这些相似用户喜欢、目标用户没喜欢过的物品。
6. 用相似用户分数累加成推荐分。
7. 排序输出 Top N。

### 4.2 UserCF 更像什么

UserCF 更像现实生活里问：

```text
和我口味差不多的人，最近还看了什么？
```

如果相似用户喜欢某个物品，而你还没接触过，系统就会把它推荐给你。

### 4.3 UserCF 的典型坑

- 用户很多时，两两计算相似度很重。
- 用户兴趣变化快，相似用户可能很快过期。
- 新用户没有行为时，无法找到相似用户。
- 容易受到小圈子行为影响。

## 5. SVD

SVD 属于矩阵分解。

它会把用户和物品都映射成低维向量：

```text
用户向量 · 物品向量 = 预测喜欢程度
```

它能学到一些不直接写在标签里的潜在兴趣，比如：

- 偏 AI 工程。
- 偏算法理论。
- 偏测试实践。
- 偏产品应用。

### 5.1 一步一步理解

原始行为矩阵像这样：

```text
        RAG  Embedding  ItemCF  SVD
小周      1      1        0      0
小林      1      1        0      0
小陈      0      0        1      1
```

SVD 想做的是，把这个很大的矩阵拆成：

```text
用户隐向量矩阵 × 物品隐向量矩阵
```

然后用：

```text
用户向量 · 物品向量 = 预测喜欢程度
```

来预测用户还没接触过的物品。

### 5.2 隐向量怎么理解

隐向量不是人工标签。

它可能学到：

- 用户偏 AI 工程。
- 用户偏推荐算法。
- 物品偏工具实践。
- 物品偏理论数学。

这些维度是从行为里学出来的，所以叫“隐含特征”。

### 5.3 SVD 的典型坑

- 不如标签推荐容易解释。
- 需要训练和调参。
- 新用户、新物品仍然有冷启动问题。
- 真实系统通常会和内容推荐、热门推荐、召回排序策略混合使用。

## 6. 四种方法怎么选

| 方法 | 看什么 | 适合 | 不适合 |
| --- | --- | --- | --- |
| 标签/内容推荐 | 物品标签和内容 | 冷启动、新物品 | 标签质量差 |
| ItemCF | 物品共现 | 行为多、物品稳定 | 新物品冷启动 |
| UserCF | 用户相似 | 社区兴趣明显 | 用户变化快 |
| SVD | 隐向量 | 稀疏矩阵、潜在兴趣 | 解释性较弱 |
