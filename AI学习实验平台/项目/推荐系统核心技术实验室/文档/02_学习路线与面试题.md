# 学习路线与面试题

## 1. 学习路线

推荐按下面顺序学：

```text
标签/内容推荐
  -> ItemCF
  -> UserCF
  -> SVD
  -> 推荐评估
  -> 混合推荐
```

原因：

- 标签推荐最直观。
- ItemCF 是协同过滤里最容易工程化的一类。
- UserCF 帮你理解相似用户。
- SVD 是矩阵分解和深度推荐的基础。

## 1.1 每个模块怎么学

### 标签/内容推荐

你要能回答：

- 物品标签从哪里来。
- 用户画像怎么构建。
- 候选物品怎么打分。
- 为什么它适合冷启动。
- 为什么它容易推荐得太窄。

### ItemCF

你要能回答：

- 什么是物品共现。
- 物品相似度怎么计算。
- 为什么要做热度归一化。
- 推荐分怎么由多个历史物品贡献。
- 为什么新物品冷启动困难。

### UserCF

你要能回答：

- 什么是相似用户。
- 用户相似度怎么计算。
- 怎么用相似用户产生候选推荐。
- UserCF 和 ItemCF 的区别。
- 用户规模大时为什么计算压力大。

### SVD

你要能回答：

- 用户-物品矩阵是什么。
- 矩阵分解在分解什么。
- 用户隐向量和物品隐向量是什么。
- 点积为什么能表示匹配程度。
- SVD 为什么解释性弱一些。

## 2. 必会面试题

### 2.1 标签推荐和协同过滤有什么区别

标签推荐看物品内容，协同过滤看用户行为。

例子：

```text
新上线一门 SVD 课程，还没人学过。
ItemCF/UserCF 因为没有行为，暂时很难推荐。
标签推荐可以根据“推荐、矩阵分解、算法”立刻推荐。
```

### 2.2 ItemCF 和 UserCF 有什么区别

ItemCF 找相似物品，UserCF 找相似用户。

ItemCF：

```text
你喜欢 A，A 和 B 相似，所以推荐 B。
```

UserCF：

```text
你和小林相似，小林喜欢 B，你没喜欢 B，所以推荐 B。
```

### 2.3 SVD 为什么有用

SVD 可以从稀疏用户行为中学习潜在兴趣。

它不只看显式标签，也能通过大量行为学出隐藏关系。

### 2.4 推荐系统为什么要评估覆盖率和多样性

如果系统永远推荐热门内容，点击率可能还可以，但用户会觉得无聊。

所以推荐系统不能只看准确率，还要看：

- Coverage：推荐覆盖了多少物品。
- Diversity：推荐列表是否多样。
- Novelty：推荐是否有新鲜感。
- Long-tail：是否能推荐长尾内容。

### 2.5 大厂常问：冷启动怎么解决

冷启动分为新用户和新物品。

新用户没有行为：

- 推荐热门内容。
- 推荐编辑精选内容。
- 根据注册信息、岗位、兴趣标签做粗推荐。
- 引导用户选择兴趣。

新物品没有行为：

- 用标签/内容推荐。
- 用文本 embedding 做内容相似。
- 给新物品一定曝光探索流量。
- 等积累行为后再进入 ItemCF/UserCF。

### 2.6 大厂常问：为什么 ItemCF 比 UserCF 更常用

很多业务里物品比用户更稳定。

用户兴趣变化快，但物品之间的相似关系相对稳定。

ItemCF 的相似度矩阵可以离线计算，在线推荐时直接读取相似物品并累加分数，工程上更稳定。

### 2.7 混合题：如果让你设计一个推荐系统，你怎么做

可以回答：

1. 冷启动阶段用热门推荐、标签推荐。
2. 用户行为积累后加入 ItemCF/UserCF。
3. 数据规模上来后加入 SVD 或深度召回。
4. 多路召回后统一排序。
5. 用 A/B 测试评估点击率、转化率、留存、覆盖率和多样性。
