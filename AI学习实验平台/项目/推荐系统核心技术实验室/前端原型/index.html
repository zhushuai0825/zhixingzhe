Recommendation Learning Lab

总览

从“相似”走向“推荐”

推荐系统不是一个算法，而是一组召回、排序和解释方法。

这一站先学四个最核心的入门模块：标签/内容推荐、ItemCF、UserCF、SVD。 它们都会输出 Top N 推荐，但“相似”的定义完全不同。

Tag内容像看物品标签和属性
ItemCF物品共现喜欢 A 的人也喜欢 B
UserCF用户相似和你相似的人喜欢什么
SVD隐向量学习用户和物品的潜在兴趣

0 基础先记住这句话

推荐系统做的事是：根据用户、物品和行为，猜“用户接下来可能喜欢什么”。标签推荐看内容，协同过滤看行为，SVD 把行为压缩成潜在兴趣空间。

和 RAG 的相似点

RAG 找“和问题最相关的证据”，推荐系统找“和用户兴趣最相关的物品”。它们都会用到相似度、Top N、召回、排序、解释和评估。

学习顺序

先从最容易解释的标签推荐开始，再进入协同过滤，最后看 SVD。

01标签/内容推荐

不需要用户群体，只要知道物品标签，就能做冷启动推荐。

02ItemCF

用用户行为统计物品相似度，是工业界非常经典的召回方法。

03UserCF

找相似用户，再看相似用户喜欢过什么，适合理解“人群相似”。

04SVD

把用户和物品映射到隐向量空间，理解矩阵分解和深度推荐的基础。

Tag / Content Based

标签/内容推荐：根据物品内容相似来推荐

它不看“别人喜欢了什么”，而是看“这个物品本身有什么标签”。

白话理解

如果你喜欢“RAG 入门”和“向量数据库”，系统会发现你偏好 AI、检索、工程这些标签，于是推荐同样带这些标签的内容。

适合场景

新物品刚上线、行为数据很少时，ItemCF/UserCF 很难工作，但标签推荐可以立刻根据内容推荐。

用户画像 = 已喜欢物品的标签累加；候选得分 = 用户画像标签 与 候选物品标签 的重合程度

标签/内容推荐完整拆解

这一模块你要学会什么

你要能从“物品有什么标签”推导到“用户喜欢什么标签”，再从用户画像推导出候选物品的推荐分。

最小数据结构

物品表至少要有 id、title、tags；用户行为表至少要有 user_id、item_id、behavior。没有行为也可以先用用户主动选择的兴趣标签。

手算顺序

先列出用户已喜欢物品，再把标签计数累加，然后逐个候选物品算标签命中，最后按分数排序。

01 数据从哪里来

每个物品要先有标签、分类、标题、正文、作者、难度等内容信息。这里先用标签做演示，真实系统还可以用文本 embedding、图片 embedding 或人工标签。

02 用户画像怎么来

把用户喜欢过的物品标签累加起来。喜欢过 3 个 AI 内容，AI 标签权重就更高；喜欢过 2 个推荐内容，推荐标签权重也会上升。

03 推荐分怎么来

候选物品的标签和用户画像越重合，分数越高。比如用户画像里 AI=3，候选物品也有 AI 标签，那它就能拿到较高贡献。

04 最大优点

新物品没有任何点击行为也能推荐，因为它只依赖内容。课程、文章、商品刚上线时，标签/内容推荐非常有用。

05 主要缺点

容易越推越窄。用户喜欢 AI，就一直推 AI，可能发现不了新兴趣。标签质量差时，推荐也会跟着变差。

06 你应该怎么看页面

先看当前用户喜欢了哪些物品，再看这些物品累加出了哪些标签，最后看推荐结果命中了哪些标签。

本模块动手验证：标签如何变成推荐

看当前用户喜欢过的物品，再看这些物品的标签如何变成用户画像，最后看候选物品为什么被推荐。

Item Collaborative Filtering

ItemCF：根据物品共现相似来推荐

ItemCF 的核心不是物品内容，而是用户行为：很多人同时喜欢 A 和 B，就认为 A 和 B 相似。

白话理解

很多学过“Embedding”的人也学“向量库”，那这两个物品就有共现关系。你喜欢 Embedding，系统可能推荐向量库。

关键公式

sim(i, j) = 喜欢 i 和 j 的用户数 / sqrt(喜欢 i 的用户数 * 喜欢 j 的用户数)

ItemCF 完整拆解

这一模块你要学会什么

你要能从用户行为表算出物品共现，再把共现变成物品相似度，最后用用户历史喜欢物品给候选物品累加推荐分。

最小数据结构

只需要 user_id、item_id、behavior。ItemCF 不关心标题语义，关键是“哪些物品被同一批用户一起喜欢”。

手算顺序

统计 count(A)、count(B)、co_count(A,B)，计算 sim(A,B)，再对目标用户的多个历史物品贡献求和。

01 数据从哪里来

ItemCF 只需要用户行为表：谁喜欢了什么、谁点击了什么、谁购买了什么。它不依赖物品标题，也不需要理解文本。

02 共现是什么意思

如果很多用户同时喜欢 A 和 B，就说 A 和 B 共现多。共现越多，通常表示两个物品越相似。

03 为什么要归一化

热门物品天然会和很多东西共现，如果不归一化，所有东西都像热门物品。除以 sqrt(count(A) * count(B)) 可以削弱热门物品的霸榜影响。

04 推荐分怎么来

看用户已喜欢物品，找到每个历史物品相似的候选，把相似度累加成推荐分。推荐分越高，候选越靠前。

05 最大优点

推荐理由清楚，工程实现稳定，适合物品相对稳定、用户行为较多的系统，比如课程、文章、商品推荐。

06 主要缺点

新物品没人喜欢过，就没有共现关系；新用户没有行为，也无法知道他喜欢什么。这就是冷启动问题。

本模块动手验证：共现如何变成推荐

看用户历史喜欢物品如何通过物品相似度，把候选物品推上推荐榜。

User Collaborative Filtering

UserCF：根据相似用户来推荐

UserCF 先找“和你兴趣相似的人”，再把他们喜欢但你没看过的东西推荐给你。

白话理解

如果你和小林都喜欢 RAG、Embedding、向量库，而小林还喜欢 Rerank，那么系统可能把 Rerank 推荐给你。

和 ItemCF 的区别

ItemCF 问“物品和物品像不像”；UserCF 问“用户和用户像不像”。两者都来自行为，不直接理解内容。

UserCF 完整拆解

这一模块你要学会什么

你要能从用户行为表算出用户相似度，再看相似用户喜欢过但当前用户没喜欢过的物品，把它们变成推荐候选。

最小数据结构

每个用户都可以表示成一个喜欢物品集合。两个集合重合越多，用户兴趣越相似。

手算顺序

列出目标用户喜欢集合，列出其他用户喜欢集合，算相似度，找相似用户贡献候选物品。

01 数据从哪里来

UserCF 也使用用户行为表，但它先比较用户和用户，而不是物品和物品。

02 用户相似怎么计算

如果两个用户喜欢过很多相同物品，就认为他们兴趣相似。这里用集合余弦相似度演示。

03 推荐分怎么来

找到相似用户后，看这些相似用户喜欢过、当前用户没喜欢过的物品。相似用户越相似，贡献越高。

04 和 ItemCF 的区别

ItemCF 更像“你喜欢的东西相似的东西”；UserCF 更像“和你相似的人还喜欢什么”。

05 最大优点

容易理解人群兴趣，适合社区、社交、兴趣圈层明显的场景。

06 主要缺点

用户数量很大时，计算用户两两相似会很重；用户兴趣变化快时，相似用户也容易过期。

本模块动手验证：相似用户如何贡献推荐

先看谁和当前用户相似，再看这些相似用户喜欢过什么候选物品。

Matrix Factorization

SVD / 矩阵分解：学习用户和物品的隐向量

SVD 不直接数标签或共现，而是把用户和物品都压缩到一个低维兴趣空间里。

白话理解

它会尝试学出一些看不见的维度，比如“偏 AI 工程”“偏算法数学”“偏产品应用”。用户和物品在这些维度上越匹配，推荐分越高。

核心直觉

用户向量 · 物品向量 = 预测喜欢程度。点积越大，表示这个物品越符合用户的潜在兴趣。

SVD / 矩阵分解完整拆解

这一模块你要学会什么

你要理解用户-物品行为矩阵很稀疏，SVD 会把用户和物品压缩成低维隐向量，再用点积预测喜欢程度。

最小数据结构

需要用户-物品评分或行为矩阵。学习阶段可以用喜欢=1、未发生=0，真实系统会区分点击、收藏、购买、评分权重。

手算顺序

先理解矩阵每一行是用户、每一列是物品，再理解用户向量和物品向量点积越大，预测兴趣越强。

01 它和 CF 最大区别

ItemCF/UserCF 直接基于共现或相似用户；SVD 会学习一组隐含维度，把用户和物品都变成低维向量。

02 隐向量是什么

隐向量不是人工写的标签，而是算法从行为矩阵里学出来的潜在兴趣。真实系统里这些维度通常不能直接命名。

03 推荐分怎么来

用户向量和物品向量做点积。点积越大，表示用户的潜在兴趣和物品的潜在属性越匹配。

04 为什么能处理稀疏

用户不可能看过所有物品，行为矩阵非常稀疏。矩阵分解能用少量维度概括用户和物品，预测没发生过的行为。

05 最大优点

能发现标签和共现之外的潜在关系，是很多经典推荐模型和深度推荐模型的基础。

06 主要缺点

解释性比标签推荐和 ItemCF 弱；训练、调参、更新也更复杂。新手先理解点积和隐向量即可。

本模块动手验证：隐向量如何产生推荐分

看当前用户隐向量，再看候选物品隐向量，理解点积为什么能表示匹配程度。
