# AI 学习实验平台

这是一个统一学习入口项目，用来承载后续所有 AI 学习系统。

当前学习边界已经收敛为：大模型应用、RAG、Agent、Embedding、相似度、向量库、Rerank、Graph RAG 和推荐算法。FastAPI、PostgreSQL、Docker、部署等工程能力只保留为历史项目或可选扩展，不作为现在这条学习路线的必学内容。

当前已经整合多个子项目：

1. `知行者AI实验室`：RAG、向量数据库、Agent、大模型应用。
2. `基于物品的协同过滤ItemCF`：推荐算法、物品相似度、Top N 推荐。
3. `标签推荐实验室`：标签画像、内容匹配、冷启动推荐。
4. `UserCF实验室`：相似用户、推荐贡献、用户协同过滤。
5. `SVD矩阵分解实验室`：用户-物品矩阵、隐向量、点积预测。
6. `推荐系统核心技术实验室`：标签推荐、ItemCF、UserCF、SVD 对比学习。
7. `Rerank重排序实验室`：向量召回后的证据重排、证据门和提准。
8. `RAG评测实验室`：评估检索命中、引用准确、回答忠实和拒答正确性。
9. `GraphRAG图谱实验室`：实体关系、多跳路径、图谱检索和证据回填。
10. `Agent工程实验室`：目标、计划、工具调用、观察、评估和反思。
11. `模型调用与Prompt工程实验室`：模型 API、Prompt、结构化输出、重试和成本。

## 推荐实验数据说明

`标签推荐实验室`、`UserCF实验室`、`SVD矩阵分解实验室`、`推荐系统核心技术实验室` 已经接入同一套本地实验数据中心。

你可以在实验台里直接：

- 新增用户，例如 `小王 / AI 学习者`。
- 新增物品，例如 `Graph RAG 实战课 / AI, RAG, 图谱`。
- 给当前用户添加喜欢行为。
- 删除当前用户的喜欢行为。
- 一键重置演示数据。

这些数据保存在浏览器 `localStorage`，不需要后端数据库。你在任意一个推荐实验室新增的数据，切到其他推荐实验室也能看到。这样做是为了让你先专心学习算法输入、计算过程、推荐解释和结果变化。

## 6. 当前整合方式

当前采用“平台壳 + 子项目 iframe”的方式。注意需要从 `AI学习实验平台` 根目录启动静态服务，这样平台前端才能访问同级的 `项目/` 目录。

优点：

- 不破坏原项目。
- 子项目仍能独立运行。
- 以后新增项目很简单。
- 文档可以按项目独立维护。

后续如果平台复杂度升高，可以再升级成统一前端路由。

## 7. 注意

新平台没有复制真实 `.env`，避免 API Key 泄露。

如果只学习当前平台里的大模型和算法实验，启动 8099 静态服务就够了。

如果以后要继续研究知行者历史后端，需要在：

```text
项目/知行者AI实验室/.env
```

按 `.env.example` 配置本地环境变量和模型 Key。
