# 平台总览

AI 学习实验平台是一个统一项目壳，用于承载多个学习型工程项目。

## 1. 当前目标

当前目标不是把所有代码揉成一个复杂系统，而是先做到：

- 一个统一入口。
- 左侧项目名称区域切换项目。
- 每个项目保留独立功能。
- 每个项目保留独立文档。
- 后续新项目可以继续接入。
- 学习重点只放在大模型、RAG、Agent 和推荐算法，不把后端、数据库、部署作为当前主线。

## 3. 当前平台能力

- 左侧品牌区项目切换。
- iframe 全屏嵌入子项目。
- 记住上一次选择的项目。
- `学习资料/RAG核心技术学习站_上篇.html` 提供纯前端交互式学习资料。
- `文档/05_学习范围与掌握标准.md` 说明当前阶段学什么、暂时不学什么、学到什么程度算够。
- `Rerank 重排序实验室` 提供向量召回后重排和证据门学习。
- `RAG 评测实验室` 提供检索、引用、回答和拒答质量评测学习。
- `Graph RAG 图谱实验室` 提供实体关系、多跳路径和原文证据回填学习。
- `Agent 工程实验室` 提供目标、计划、工具调用、观察、评估和反思学习。
- `模型调用与 Prompt 工程实验室` 提供模型 API、结构化输出、重试和成本学习。
- `推荐系统核心技术实验室` 提供纯前端推荐算法对比学习。
- `标签推荐实验室`、`UserCF实验室`、`SVD矩阵分解实验室` 把推荐算法拆成独立学习模块，适合逐个深入。
- `文档/04_全平台质量验收清单.md` 用于每次修改后的完整检查。

## 3.1 推荐实验数据中心

12、13、14 四个推荐实验室和推荐系统总览共用一份本地实验数据，数据保存在浏览器 `localStorage`。

它支持：

- 新增用户。
- 新增物品和标签。
- 给当前用户新增喜欢行为。
- 删除当前用户已有喜欢行为。
- 重置回默认演示数据。

这几个实验室不是静态假页面。你新增一个物品，给用户添加喜欢后，标签推荐、UserCF、SVD 和推荐系统总览里的推荐结果都会重新计算。这样你可以像做实验一样观察：输入数据变了，用户画像、相似用户、隐向量和 Top N 推荐为什么会跟着变。

## 4. 为什么不直接改成一个大系统

因为现在学习项目还在快速增长。

如果太早合成一个大前端，会带来：

- 路由复杂。
- 样式冲突。
- 模块边界混乱。
- 文档混乱。
- 修改一个项目影响另一个项目。

当前方案更适合学习阶段：

```text
先统一入口，再逐步统一能力。
```
