# 全平台质量验收清单

这份清单用于每次修改 AI 学习实验平台后做收尾检查。目标不是追求形式，而是确认项目真的能学、能试、能解释、能复现。

## 1. 平台入口检查

启动平台：

```bash
cd /Users/zhushuai/Downloads/学习/AI学习实验平台
python3 -m http.server 8099
```

打开：

```text
http://127.0.0.1:8099
```

必须满足：

- 能自动跳转到 `平台前端/`。
- iframe 能正常加载默认项目。
- 刷新页面后仍能打开上次选择的项目。
- 子项目左上角项目名称可以打开切换菜单。
- 切换到每个项目后页面不空白。

## 3. RAG 主线验收

文本切分实验室必须能验证：

- 修改 chunk size 后，切片数量或切片内容会变化。
- 修改 overlap 后，相邻 chunk 的重叠内容会变化。
- 过程说明能讲清“为什么这样切”。

Embedding 与相似度实验室必须能验证：

- 查询文本会变成查询向量。
- 文档文本会变成文档向量。
- 画布能展示查询向量和候选向量的距离或连线。
- 能解释为什么第一条最相似。

向量库实验室必须能验证：

- 可以看到 collection。
- 可以 upsert 数据。
- 可以 query Top K。
- 可以删除数据。
- 可以查看 metadata。
- 可视化能表现命中关系。

Rerank 重排序实验室必须能验证：

- 可以修改问题。
- 可以调整向量召回 Top K。
- 可以调整最终保留 Top N。
- 可以切换 Rerank 策略。
- 可以看到向量库初筛顺序。
- 可以看到 Rerank 后证据顺序。
- 可以点击候选查看分数解释。
- 可视化能展示排序前后名次移动。
- 证据门能提示证据是否足够进入 prompt。

RAG 评测实验室必须能验证：

- 可以选择评测问题。
- 可以新增评测问题和标准证据。
- 可以调整检索 Top K。
- 可以切换系统版本。
- 可以看到检索结果、系统回答和标准答案。
- 可以计算 Recall@K、MRR、引用准确率、忠实度、答案覆盖和拒答正确性。
- 可以根据短板输出失败诊断。
- 指标可视化能展示质量短板。

Graph RAG 图谱实验室必须能验证：

- 可以修改问题。
- 可以调整最大跳数。
- 可以新增实体。
- 可以新增关系。
- 可以新增证据片段。
- 可以看到命中实体和多跳路径。
- 可以看到路径关联的原文证据。
- 3D 图谱能旋转、缩放、悬停和点击。
- 页面能解释 Graph RAG 和普通向量 RAG 的区别。

Agent 工程实验室必须能验证：

- 可以修改目标。
- 可以切换 Agent 类型。
- 可以开关 RAG、Graph RAG 和反思检查。
- 可以调整最大工具调用步数和证据阈值。
- 可以新增知识片段和工具。
- 可以看到计划、工具调用、观察、评估、行动和反思轨迹。
- 可以点击每个步骤查看输入、输出和解释。
- 执行轨迹图能展示 Agent 状态流。

模型调用与 Prompt 工程实验室必须能验证：

- 可以修改 system prompt、用户问题和上下文。
- 可以切换模型策略、温度和最大输出。
- 可以开关 JSON 输出、引用校验和自动重试。
- Token、成本和延迟估算会随输入变化。
- Prompt 诊断能指出缺少 JSON 约束、引用规则或上下文等问题。
- 调用链路图能展示 Prepare、Request、Generate、Validate、Retry 等步骤。

知行者 AI 实验室必须能验证：

- 上传文档后能看到切片。
- 能看到向量化或索引状态。
- 提问后能看到命中的引用来源。
- Agent 实验室能展示计划、工具调用、观察、评估、最终结果。

- 有 `实验数据中心`。
- 可以新增用户。
- 可以新增物品。
- 可以给当前用户添加喜欢。
- 可以删除当前用户喜欢。
- 可以重置演示数据。
- 标签推荐、UserCF、SVD、推荐系统总览共用同一份浏览器本地数据。

## 5. 文档验收

每个项目至少要能从文档里回答：

- 这个项目学什么。
- 页面怎么打开。
- 数据从哪里来。
- 用户可以自己改什么数据。
- 算法或流程怎么算。
- 怎么判断结果对不对。
- 常见坑是什么。
- 面试应该怎么讲。

推荐的文档结构：

```text
README.md
文档/00_文档索引.md
文档/01_学习手册.md
文档/02_工程实现说明.md
文档/03_测试验收清单.md
```

小实验室可以合并文档，但不能缺少“怎么动手验证”。

## 7. 质量标准

一个模块算“完成”，至少满足：

- 页面不是静态展示，用户能自己输入或修改数据。
- 结果不是固定文案，输入变了结果会变。
- 页面能解释中间过程，而不是只给最终结果。
- 有可视化帮助理解。
- 有文档和验收清单。
- 刷新页面后不会出现明显报错、空白、旧缓存、错别字导致的理解障碍。

如果一个模块只是演示，不接真实后端或真实模型，必须在页面或文档里写清楚“这是教学版”，避免把教学算法误解成生产算法。
