# 运行说明

这份文档说明 AI 学习实验平台和子项目怎么启动。

当前学习大模型和算法时，只需要启动平台静态服务。知行者、ItemCF、Embedding、文本切分、向量库里曾经保留过后端能力，但那是可选历史扩展；如果你的目标是先学 RAG、Agent、Embedding、相似度和推荐算法，不需要先启动 FastAPI、PostgreSQL 或 Docker。

## 1. 启动平台入口

```bash
cd /Users/zhushuai/Downloads/学习/AI学习实验平台
python3 -m http.server 8099
```

打开：

```text
http://127.0.0.1:8099
```

## 2. 运行知行者 AI 实验室

知行者前端在平台 iframe 中直接打开。

如果只看静态页面，启动平台即可。

如果以后要研究历史真实 RAG 和 Agent 后端，再启动基础设施和后端。

### 2.1 启动数据库和向量库

```bash
cd /Users/zhushuai/Downloads/学习/AI学习实验平台/项目/知行者AI实验室
docker compose up -d
```

### 2.2 配置环境变量

复制示例：

```bash
cp .env.example .env
```

然后按需填写 API Key。

### 2.3 启动后端

```bash
cd /Users/zhushuai/Downloads/学习/AI学习实验平台/项目/知行者AI实验室/后端
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --host 127.0.0.1 --port 8010
```

健康检查：

```bash
curl http://127.0.0.1:8010/health
```

## 12. 常见问题

### 12.1 平台能打开，但子项目空白

检查子项目路径是否存在：

```text
项目/知行者AI实验室/前端原型/index.html
项目/基于物品的协同过滤ItemCF/前端原型/index.html
项目/Embedding与相似度实验室/前端原型/index.html
项目/文本切分实验室/前端原型/index.html
```

### 12.2 知行者后端不通

检查：

```bash
curl http://127.0.0.1:8010/health
```

如果 Chroma 失败，确认：

```text
CHROMA_HOST=127.0.0.1
CHROMA_PORT=8001
```

### 12.3 ItemCF 后端不通

检查：

```bash
curl http://127.0.0.1:8020/health
```

如果没有启动后端，ItemCF 前端仍可使用本地算法模式。

### 12.4 Embedding 后端不通

检查：

```bash
curl http://127.0.0.1:8030/health
```

如果没有启动后端，Embedding 前端仍可使用离线教学算法。

### 12.5 文本切分后端不通

检查：

```bash
curl http://127.0.0.1:8040/health
```

如果没有启动后端，文本切分前端仍可使用离线教学算法。
