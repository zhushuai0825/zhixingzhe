# 项目接入规范

以后新项目接入 AI 学习实验平台，按这份规范来。

## 1. 推荐目录结构

每个子项目放在：

```text
项目/<项目名称>/
```

内部建议结构：

```text
项目名称/
  README.md
  前端原型/
    index.html
    styles.css
    app.js
  后端/
    README.md
    requirements.txt
    app/
  文档/
    00_文档索引.md
  需求/
  规划/
```

## 2. 必须有的文件

每个项目至少要有：

- `README.md`
- `前端原型/index.html`
- `文档/00_文档索引.md`

如果有后端，还要有：

- `后端/README.md`
- `后端/requirements.txt`

## 3. 平台前端怎么接入

修改：

```text
平台前端/app.js
```

在 `projects` 里增加一项：

```js
newProject: {
  url: "../项目/新项目/前端原型/index.html",
}
```

当前平台壳只负责 iframe 加载，不在 `平台前端/index.html` 里放顶部菜单。项目切换入口在每个子项目左上角，由子项目发送消息给平台：

```js
window.parent?.postMessage(
  { type: "learning-platform:switch-project", project: "newProject" },
  "*"
);
```

所以新增项目时，还要同步修改子项目的项目切换菜单。推荐系统相关纯前端项目共用：

```text
项目/推荐系统核心技术实验室/前端原型/recommendation-shared.js
```

如果是独立项目，例如知行者、ItemCF、Embedding、文本切分、向量库，则需要在它自己的 `前端原型/index.html` 或对应共享脚本里加入项目按钮。

接入后必须验证：

```text
点击子项目左上角项目名称
  -> 选择新项目
  -> iframe 跳转到新项目
  -> 刷新后仍能记住最后选择的项目
```

## 4. 端口规范

建议端口：

| 项目类型 | 端口 |
| --- | --- |
| 平台前端 | 8099 |
| 知行者后端 | 8010 |
| ItemCF 后端 | 8020 |
| Embedding 与相似度后端 | 8030 |
| 文本切分后端 | 8040 |
| 向量库后端 | 8050 |
| 新 AI 应用后端 | 8060 起 |

避免多个项目抢同一个端口。

## 5. 环境变量规范

不要把真实 `.env` 提交或复制到平台文档里。

每个项目只保留：

```text
.env.example
```

真实配置由本地自己维护。

## 6. 文档规范

每个项目文档至少包括：

- 文档索引。
- 系统使用手册。
- 学习手册。
- API 说明。
- 运行部署与排错。
- 测试验收清单。
- 后续迭代计划。

小项目可以合并，但必须能回答：

- 这个项目学什么？
- 怎么运行？
- 怎么使用？
- 怎么验证？
- 后续怎么扩展？

## 7. 前端实验室最低交互标准

后续新增实验室，不建议只做静态说明页。至少要做到：

- 有输入区：用户可以输入文本、参数、用户行为或样例数据。
- 有过程区：展示中间计算过程，例如 chunk、embedding、相似度、候选、贡献分。
- 有结果区：展示最终 Top K、Top N、回答、推荐或任务结果。
- 有解释区：用口语化方式说明“为什么是这个结果”。
- 有重置能力：方便反复实验。
- 有文档：说明输入是什么、输出是什么、怎么验证。

如果是推荐系统类项目，必须能新增或修改用户行为；如果是 RAG 类项目，必须能新增或修改文档/问题/检索参数。
