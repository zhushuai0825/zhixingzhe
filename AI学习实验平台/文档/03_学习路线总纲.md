# 学习路线总纲

这个平台会逐步承载你的 AI 应用开发学习路线。

当前路线只聚焦大模型应用和算法：RAG、Embedding、向量库、Rerank、Graph RAG、Agent、Prompt 工程、推荐算法。FastAPI、数据库、部署、云服务器暂时不作为学习主线，避免你被工程杂项分散注意力。

## 1. 推荐学习顺序

先按这条主线学 RAG，不要一开始就跳到完整系统：

```text
文本切分实验室
  -> Embedding 与相似度实验室
  -> 向量库实验室
  -> Rerank 实验室
  -> RAG 评测实验室
  -> Graph RAG 图谱实验室
  -> Agent 工程实验室
  -> 模型调用与 Prompt 工程实验室
  -> 知行者 AI 实验室
```

扩展路线：

```text
ItemCF 学习实验室
  -> 标签推荐实验室
  -> UserCF 实验室
  -> SVD 矩阵分解实验室
  -> 推荐系统核心技术实验室
```

ItemCF 和推荐系统不是 RAG 主线，但它们能帮助你理解“相似度、Top N、召回、排序、解释”的另一类应用。

配套 HTML 学习站：

```text
学习资料/RAG核心技术学习站_上篇.html
```

上篇已经覆盖：文本切分、Embedding、向量库、Rerank、RAG 评测、Graph RAG、Agent、模型调用与 Prompt 工程，并补充了综合面试题。

## 2. 第一阶段：RAG 基础链路

项目：

- 文本切分实验室。
- Embedding 与相似度实验室。
- 向量库实验室。
- Rerank 重排序实验室。
- RAG 评测实验室。
- Graph RAG 图谱实验室。
- Agent 工程实验室。
- 模型调用与 Prompt 工程实验室。

要学会：

- 文档如何切片。
- Embedding 如何工作。
- 向量数据库做什么。
- 查询文本如何命中 chunk。
- chunk、向量、metadata 如何串起来。
- Rerank 如何把初筛证据重新排序。
- 证据不足时为什么不能硬生成。
- Recall@K、MRR、引用准确率和忠实度如何判断 RAG 质量。
- Graph RAG 如何通过实体、关系和多跳路径补充普通向量检索。
- 为什么图谱路径仍然要回到原文 chunk 做证据引用。
- Agent 如何把 RAG、Graph RAG、Rerank 组织成可控工具调用流程。
- 模型调用如何处理 Prompt、结构化输出、重试、成本和日志。

推荐你按下面的小目标检查自己是否学会：

| 模块 | 学完后你要能说清 | 动手验证 |
| --- | --- | --- |
| 文本切分 | 为什么不能把整篇文档直接塞给模型 | 改 chunk size 和 overlap，看切片怎么变 |
| Embedding | query 和 chunk 为什么要进入同一个向量空间 | 输入问题，看查询向量和候选向量的距离 |
| 向量库 | collection、upsert、query、metadata 分别是什么 | 新增数据、查询 Top K、删除数据 |
| Rerank | 为什么向量召回第一名不一定是最好证据 | 看候选从初筛顺序到重排顺序怎么移动 |
| RAG 评测 | 为什么“能回答”不等于“答得准” | 看 Recall@K、MRR、忠实度和拒答正确性 |
| Graph RAG | 为什么复杂问题要看实体关系路径 | 新增实体和关系，看多跳路径怎么影响证据 |
| Agent 工程 | 为什么多步任务需要状态、工具和停止条件 | 关闭某个工具，看轨迹和最终结果怎么变化 |
| 模型调用 | 为什么真实系统要校验输出和控制成本 | 清空上下文或删掉 JSON 约束，看校验和重试怎么变化 |

这条线学完，你不是只会调一个 RAG demo，而是能解释一个 RAG 系统从数据进入、检索、排序、生成到评测的完整闭环。

## 3. 第二阶段：Agent 与模型调用

项目：

- Agent 工程实验室。
- 模型调用与 Prompt 工程实验室。
- 知行者 AI 实验室。

要学会：

- Agent 如何调用工具。
- Agent 如何维护状态、选择工具、评估证据和停止运行。
- Prompt、结构化输出、模型调用日志、失败重试和成本控制。
- 大模型如何基于引用回答，而不是脱离证据自由发挥。
- 一个 AI 应用如何把“检索、重排、评估、生成、反思”串成闭环。

## 4. 第三阶段：看懂推荐算法

项目：

- 基于物品的协同过滤 ItemCF。
- 标签推荐实验室。
- UserCF 实验室。
- SVD 矩阵分解实验室。
- 推荐系统核心技术实验室。

要学会：

- 标签/内容推荐。
- 用户行为矩阵。
- 物品共现。
- UserCF。
- SVD / 矩阵分解。
- 物品相似度。
- 推荐分。
- Top N 推荐。
- 推荐算法如何解释、评估和对比。

建议学习顺序：

```text
标签推荐实验室
  -> ItemCF 学习实验室
  -> UserCF 实验室
  -> SVD 矩阵分解实验室
  -> 推荐系统核心技术实验室
```

原因：

- 标签推荐最容易理解，先学“用户画像”和“内容匹配”。
- ItemCF 用行为看物品相似，是协同过滤里最经典的一支。
- UserCF 再反过来看“用户和用户像不像”。
- SVD 最后学，因为它需要先理解矩阵、稀疏、向量和点积。
- 推荐系统核心技术实验室用于最后复盘四种方法的区别。

## 5. 第四阶段：真实模型能力复盘

这阶段不急着新建后端系统，而是把前面模块里学到的模型能力串起来：

- 本地模型和 API 模型差异。
- 中文 Embedding 选择。
- 文本切分如何影响检索。
- 向量库如何保存 chunk embedding。
- Collection、Upsert、Query、Metadata、Delete。
- Rerank 提升证据质量。
- 模型延迟和成本。

## 6. 第五阶段：Agent 深入

- 工具定义。
- JSON schema。
- 状态流。
- 多步任务。
- 失败重试。
- 停止条件。
- Agent 评测。

这部分已经由 `Agent 工程实验室` 和 `模型调用与 Prompt 工程实验室` 承担。后续如果继续扩展，优先补“多工具协作、长期记忆、任务分解和 Agent 评测”，不要先跳到部署。

## 7. 学习原则

不要只堆技术名词。

每学一个技术，都要在项目里回答：

- 它解决什么问题？
- 输入是什么？
- 输出是什么？
- 数据存在哪里？
- 怎么验证它是对的？
- 失败时怎么排查？
